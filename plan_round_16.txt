练习 2：让 POST /run 从“回显”改成“真处理”

在 api.py 里把 POST /run 改造成这种结构：

from fastapi import FastAPI, HTTPException
from schemas import RunRequest
from io_utils import read_records, write_records
from core import filter_records, dedup_records, build_summary
from db import init_db, save_run
from datetime import datetime

app = FastAPI()

@app.post("/run")
async def run_job(payload: RunRequest):
    try:
        records = read_records(payload.input, payload.format)
    except FileNotFoundError:
        raise HTTPException(status_code=404, detail="input file not found")
    except ValueError as e:
        raise HTTPException(status_code=400, detail=str(e))

    raw_count = len(records)
    filtered = filter_records(records, payload.keyword)
    filtered_count = len(filtered)
    final_records = dedup_records(filtered, payload.format) if payload.dedup else filtered
    final_count = len(final_records)

    write_records(final_records, payload.output, payload.format)

    summary = build_summary(
        payload.input,
        payload.output,
        raw_count,
        filtered_count,
        final_count
    )

    init_db()
    save_run(
        payload.input,
        payload.output,
        raw_count,
        filtered_count,
        final_count,
        datetime.now().isoformat()
    )

    return {"summary": summary}

这一题的关键不是代码长短，而是你要真正完成这条链：

请求体 → 核心逻辑 → 输出文件 → SQLite → 响应

FastAPI 官方对请求体和错误处理的说明，正好支持这种写法；Python sqlite3 官方文档也明确建议对 SQL 使用占位符参数绑定，而不是字符串拼接。

在 api.py 里加一个新接口：

from fastapi import UploadFile, File, Form
from pathlib import Path

@app.post("/run-upload")
async def run_upload(
    file: UploadFile,
    format: str = Form(...),
    keyword: str | None = Form(None),
    dedup: bool = Form(False)
):
    temp_dir = Path("input/uploads")
    temp_dir.mkdir(parents=True, exist_ok=True)

    temp_path = temp_dir / file.filename
    content = await file.read()
    temp_path.write_bytes(content)

    output_path = Path("output") / f"uploaded_{file.filename}"

    records = read_records(temp_path, format)
    raw_count = len(records)
    filtered = filter_records(records, keyword)
    filtered_count = len(filtered)
    final_records = dedup_records(filtered, format) if dedup else filtered
    final_count = len(final_records)

    write_records(final_records, output_path, format)

    summary = build_summary(
        str(temp_path),
        str(output_path),
        raw_count,
        filtered_count,
        final_count
    )

    init_db()
    save_run(
        str(temp_path),
        str(output_path),
        raw_count,
        filtered_count,
        final_count,
        datetime.now().isoformat()
    )

    return {"summary": summary}

这里先用最朴素的方式把上传文件落到本地临时目录，再复用现有逻辑。pathlib 非常适合这种临时路径管理；FastAPI 官方文件上传文档也明确表明 UploadFile 是文件接口的核心入口之一。 

练习 10：故意触发一个上传错误

例如：
	•	上传空文件
	•	format 写错
	•	上传内容和 format 不匹配

然后给这些情况补最小错误处理。
目标不是把所有边界全做完，而是先把常见错误从“裸异常”变成更可理解的 HTTP 响应。FastAPI 官方建议用 HTTPException 来处理这类错误响应。

from fastapi.testclient import TestClient
from api import app

client = TestClient(app)

def test_health():
