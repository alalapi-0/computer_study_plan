# Round 08 · 升级路线速查

## Web UI 完成路径

1. 点击 `r08-fin-comp` 的“运行”，确认输出显示 Round 08 的项目收口、SQLite 和 API 排练检查结果。
2. 打开本文件“阅读”，按下面的小抄判断下一阶段路线。
3. 能说清楚测试、持久化和服务化边界后，手动完成 `r08-fin-sheet` 和 `r08-fin-acc1`。

## 建议顺序

1. 先走路线 A（工程化和测试基础）。
2. 再走路线 B（服务化与 API 设计）。
3. 最后走路线 C（AI/ML 框架实战）。

## 每条路线开始前自检

- 是否有可运行的最小项目骨架。
- 是否有至少一个可重复执行的验证命令。
- 是否能解释“为什么现在进入这条路线”。

## 三条路线的选择标准

| 路线 | 进入条件 | 本仓库后续对应 |
|---|---|---|
| A 工程化 | 已有最小项目、最小测试、清晰 README | Round 09 起 |
| B 服务化 | 能解释 HTTP 请求/响应，已有持久化记录 | Round 14 起 |
| C AI/ML | 能稳定处理数据文件，并能复现实验记录 | Round 18 起 |

## 本轮验收自问

- 测试失败时，我能不能只看输出定位到哪个函数坏了？
- SQLite 里保存的是原始样本，还是运行摘要？
- 服务接口返回的是“已经处理完成”，还是“已接受任务”？
- 为什么本轮 Web UI 不直接安装 FastAPI 并启动服务？
