## HTTP 方法速查

| 方法 | 一句话语义 | 常见状态 |
|---|---|---|
| `GET` | 读取资源，不应该改变服务端状态 | `200` / `404` |
| `POST` | 提交数据或创建任务 | `201` / `400` |
| `PUT` | 完整替换资源 | `200` / `204` / `404` |
| `PATCH` | 局部修改资源 | `200` / `400` / `404` |
| `DELETE` | 删除资源 | `204` / `404` |

## ai_prep_tool 路由草图

| 方法 | 路径 | 用途 | 响应 |
|---|---|---|---|
| `GET` | `/health` | 看服务是否可用 | `{"status": "ok"}` |
| `GET` | `/runs` | 查询历史运行记录 | `{"runs": [], "total": 0}` |
| `GET` | `/runs/{run_id}` | 查询某次运行详情 | 成功 `200`，不存在 `404` |
| `POST` | `/run` | 提交一次处理任务 | 成功 `201`，参数错 `400` |

## JSON 合同口诀

1. 字段名稳定。
2. 类型稳定。
3. 成功响应和错误响应都要设计。
4. 错误响应至少有 `error` 和 `message`。
5. 路径参数、查询参数、请求体不要混在一起讲。

## Round 14 到 Round 15 的连接

Round 14 只做合同和 mock API，不安装 FastAPI，也不启动真实服务。Round 15 再把这些接口迁移成 FastAPI 路由，并通过 `/docs` 做交互式测试。
