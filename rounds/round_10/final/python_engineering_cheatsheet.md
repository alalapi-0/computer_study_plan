## 文件职责速查

| 文件 | 职责 | 不应该做什么 |
|---|---|---|
| `cli.py` | 解析参数、组织入口 | 不堆业务处理细节 |
| `core.py` | 过滤、去重、统计等核心逻辑 | 不直接读写真实文件 |
| `io_utils.py` | 读输入、写输出、创建目录 | 不做复杂业务决策 |
| `config.py` | 读取 `config.ini` 并提供 typed getter | 不把配置值散落到各处 |
| `log_utils.py` | 初始化日志格式、级别和文件 | 不在每个模块重复配置 logging |
| `errors.py` | 定义可控的领域错误 | 不写入口流程 |
| `ai_prep_tool.py` | 薄入口，串联模块并捕获用户错误 | 不把所有逻辑重新堆回来 |

## 配置与日志口诀

- 配置用 fallback：`config.get(..., fallback=...)`。
- 用户摘要用 `print()`，排查过程用 `logging`。
- 日志写入 `logs/app.log`，输出写入 `output/`。
- `output/` 和 `logs/` 是运行副产物，不应该提交。

## 错误处理口诀

- 核心函数发现问题：抛 `PrepToolError` 或它的子类。
- 入口捕获问题：打印 `error: ...` 并返回非 0 状态码。
- 不在核心函数里 `sys.exit()`。
- 缺失输入、空输入、格式不支持是最常见的显式错误。

## 最小通过标准

- 沙盒项目不再依赖单文件堆叠全部逻辑。
- 配置与日志职责分离，用户输出与排查日志可区分。
- 核心纯函数具备最小自检或可断言样例。
- 缺失输入等常见错误不会出现长 traceback，而是给出可读错误。
- `if __name__ == "__main__"` 只负责把 `main()` 的返回码交给系统。
