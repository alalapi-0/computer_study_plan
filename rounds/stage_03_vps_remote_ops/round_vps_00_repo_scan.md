# Round VPS-00 · 扫描仓库与生成治理报告

## 概览

| 项目 | 内容 |
|---|---|
| 主题 | 仓库结构扫描与治理建议 |
| 操作权限等级 | **Level 0** |
| 是否需要用户授权 | 否 |
| 前置 | 无（任何时候都可以重新执行） |
| 下一轮 | [VPS-01](round_vps_01_repo_cleanup.md) |

## 目标

- 完整列出当前仓库结构。
- 识别**重复 / 过期 / 命名混乱 / 残留**等待清理项。
- 给出**只读**的治理建议清单（不删不改）。
- 输出供 VPS-01 实际执行的"拟动作清单"。

## 背景

`computer_study_plan` 仓库中已经存在：

- 22 份主线 Round 概览（`round_00.md` ~ `round_21.md`）。
- 22 份计划文本源（`plan_round_00.txt` ~ `plan_round_21.txt`）。
- Round 00 的最小可运行闭环。
- 5 份长期规划文档（`docs/`）。

要把"远程 VPS 操作"接入总控之前，必须先**对仓库做一次完整扫描**，弄清楚：哪些是真正的重复、哪些只是"看起来像"、哪些必须保留。

## 涉及文件

仅**只读**；本轮**不修改**任何文件。

## 操作权限等级

- 等级：**Level 0**
- 行为限制：仅文档分析、仅生成报告、不连接远程服务器、不删除任何文件。

## 具体任务

1. 列出根目录全部文件与子目录。
2. 列出 `docs/`、`rounds/` 子目录全部文件。
3. 检查是否存在以下五类潜在问题：
   - 重复内容
   - 过期文档
   - 临时残留
   - 命名混乱
   - 与项目无关
4. 对照 `docs/governance/file_naming_rules.md` 检查命名一致性。
5. 对照 `docs/governance/repo_rules.md` 中的高保护对象清单，识别"默认禁止删除"对象。
6. 输出**结构摘要**与**治理建议**，提交给 VPS-01 作为输入。

## 验收标准

- [ ] 已生成结构摘要：根目录、`docs/`、`rounds/` 三层结构清晰可见。
- [ ] 已列出"疑似重复 / 过期 / 命名混乱 / 残留 / 与项目无关"五类清单（哪怕为空也要明确标注）。
- [ ] 已识别高保护对象（不可删除）。
- [ ] 给出 VPS-01 可直接执行的拟动作清单。
- [ ] 本轮未修改任何文件。

## 风险与禁止事项

- 不修改任何文件。
- 不删除任何文件。
- 不连接远程服务器。
- 不引入新依赖。
- 不基于猜测删除"看起来重复"的文件。

## 输出物

- 仓库结构摘要（粘贴在变更报告中或写入本 Round 旁的 `outputs/` 目录）。
- 治理建议清单（拟合并 / 拟删除 / 拟重命名）。
- 高保护对象清单。
