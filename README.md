# 🚀 计算机学习计划（零基础入门指南）

> **特别适合**：想学编程但不知道从哪开始的初学者、对命令行感到恐惧的AI使用者、想系统掌握计算机基础的非科班同学

一套面向 AI 应用方向的计算机基础学习路线，通过**把大目标拆解成可交付的小任务、快速迭代、及时正反馈**来推动持续学习。

## ✨ 给零基础同学的特别说明

如果你是编程新手，请先阅读这部分：

**“我完全不懂编程，能用这个计划吗？”** → **当然可以！**

这个计划就是为**零基础**设计的，我们从头开始：
1. **从双击鼠标开始**：先学会在电脑上打开文件
2. **从命令行开始**：先学7个最基础的命令
3. **从小任务开始**：每次只学一个概念，立即练习

**“看到代码我就头晕”** → **这正是我们要解决的问题！**

我们会用游戏化的方式学习：
- ✅ 每个任务都很小（5-15分钟就能完成）
- ✅ 完成一个，自动打勾一个
- ✅ 即时看到自己的进度
- ✅ 不会卡在某个地方很久

**“怎么知道自己学会了？”** → **有明确的验收标准！**

每个阶段都有清晰的“完成”标准：
- [ ] 我能在命令行里创建文件夹

## 🎯 设计理念

## 设计理念

### 核心学习模型

本计划采用"读 → 做 → 验"三步循环：

1. **读**：精读一篇推荐文档，建立概念
2. **做**：完成脚本练习（指导和参考答案写在注释里）
3. **验**：对照验收标准，确认真正掌握了

### 为什么这样设计

- **可交付的小任务** → 每个练习都有明确的完成标准，减少"完成了没有"的焦虑
- **快速迭代** → 每次练习结果即时可见（命令行/代码直接运行）
- **正反馈闭环** → 看到脚本跑起来、看到测试通过，就是最好的进度感
- **渐进式拓展** → 每轮建立在上一轮基础上，不跳步，不遗漏

## 📊 进度系统（自动打勾系统）

这个计划最棒的地方是：**你不用手动记录进度**。系统会自动帮你打勾 ✅！

### 🎯 进度系统工作原理

就像玩游戏时的成就系统一样：

| 文件 | 作用 | 比喻 |
|------|------|------|
| `progress.json` | **唯一的真实记录**，保存所有任务状态 | 游戏存档文件 |
| `progress_data.js` | `progress.json` 的浏览器版本 | 存档的云端同步版 |
| `mark_done.sh` | 完成任务的“打卡工具” | 点击“完成任务”按钮 |
| `progress.html` | 可视化的进度看板 | 游戏成就界面 |

### 🎮 如何使用进度系统

#### **方式一：最简单的方式（推荐给新手）**
1. **运行练习脚本**：每个 `exercises.sh` 都会自动帮你标记完成
2. **刷新看板**：双击 `progress.html` → 按 `⌘R` 刷新
3. **完成！** ✅ 自动出现在看板上

#### **方式二：手动标记（用于阅读任务）**
有些任务是“阅读文档”，没有自动脚本。这时需要手动标记：

1. 打开终端，进入项目文件夹：
   ```bash
   cd /Users/alalapi/Desktop/computer_study_plan
   ```

2. 查看所有任务ID：
   ```bash
   bash mark_done.sh
   ```
   你会看到类似这样的列表：
   ```
   ✅ 已完成 (0/20):
   ⬜ 未完成 (20):
      w1-read
      w1-ex1
      w1-ex2
      ...
   ```

3. 标记阅读任务完成：
   ```bash
   # 比如，你读完了 week1 的 notes.md
   bash mark_done.sh w1-read
   ```

4. 刷新 `progress.html` 看板，看到 ✅

#### **方式三：取消标记（万一标记错了）**
```bash
# 取消某个任务的完成标记
bash mark_done.sh w1-read --undo
```

### 🌐 两种打开看板的方式

#### **方法1：双击打开（最简单）**
1. 在“访达”里找到 `progress.html`
2. 双击它
3. 浏览器打开后，按 `⌘R` 刷新看进度变化

> ⚠️ **注意**：这种方式需要你手动刷新（`⌘R`）才能看到最新进度

#### **方法2：本地服务器打开（自动刷新）**
1. 在终端里输入：
   ```bash
   cd /Users/alalapi/Desktop/computer_study_plan
   python3 -m http.server 8000
   ```
2. 打开浏览器，访问：`http://localhost:8000/progress.html`
3. 看板会**自动刷新**，完成练习后立即看到变化

> 💡 **建议**：刚开始用方法1，熟悉后用方法2（体验更好）

### 🎯 任务ID命名规则

所有任务ID都遵循相同的模式，容易记忆：

| 任务ID | 含义 | 例子 |
|--------|------|------|
| `w1-read` | 第1周阅读任务 | 阅读 `week1/notes.md` |
| `w1-ex1` | 第1周练习1 | 练习脚本的第1个练习 |
| `w1-self` | 第1周自测 | 周总结和自我测试 |
| `fin-comp` | 综合练习 | 最终的综合性大作业 |

**小技巧**：记不住任务ID时，直接运行 `bash mark_done.sh` 查看列表。

## 约束条件（Constraints）

### 内容约束

1. **明确的"不学什么"** — 每个 Round 都有"本轮不学什么"一节，防止一次学太多，迷失方向
2. **只在有效前置基础上推进** — 每个 Round 都有 `前置` 字段，不允许跳轮次
3. **练习代码可直接运行** — 所有代码示例必须是可以直接 `python script.py` 运行的完整代码
4. **每轮只聚焦一个主题** — 每个 Round 的主题概览表限制在 3-5 个关键技术
5. **验收标准优先于知识量** — 一个 Round 的价值不在"讲了多少"，而在"能验收多少"

### 难度约束

- ⭐☆☆☆☆ = 极简入门
- ⭐⭐☆☆☆ = 基础
- ⭐⭐⭐☆☆ = 需要练习才能掌握
- ⭐⭐⭐⭐☆ = 有一定复杂度
- ⭐⭐⭐⭐⭐ = 高难度

> Round 07 之后不建议快速跳过，每个 Round 需要 3 周真实练习时间。

### 学习节奏约束

- **每周约 8 小时** — 是平均参考，不是硬性限制
- **3 周 1 轮** — 不建议压缩成 1 周，给练习留足时间
- **不跳轮次** — 哪怕感觉简单，也应该把练习做完，"能运行"和"真正理解"之间有差距

### 工具/库约束

| 阶段 | 推荐使用 | 暂不使用 |
|------|---------|---------|
| Round 00-06 | 系统命令行工具、vim/nano | GUI IDE、第三方工具 |
| Round 07-13 | Python 标准库、pathlib、sqlite3、argparse | pandas、ORM、第三方 CLI 框架 |

如果你是第一次接触编程、命令行，请严格按以下步骤操作：

**📋 第0步：准备工作**
1. 确保你有一台电脑（macOS 或 Linux）
2. 打开这个项目文件夹（就是你现在看README的地方）
3. 如果你在macOS上，打开“访达”（Finder），找到 `computer_study_plan` 文件夹

**🖥️ 第2步：打开“终端”（Terminal）**
- **macOS用户**：按 `⌘ + 空格`，输入“终端”，按回车
- **Linux用户**：按 `Ctrl + Alt + T`

你会看到一个黑色或白色的窗口，里面可能有一行文字，最后有个闪烁的光标。**这就是“命令行界面”（CLI）**，也是你要学习的地方。

**📖 第3步：阅读第一轮学习材料**
1. 在“访达”里找到并双击打开 `round_00.md`（这是第0轮的概览）

3. 然后进入 `rounds/round_00/week1/` 文件夹
4. 双击打开 `notes.md`，这是第一周的学习笔记

**🎮 第4步：开始你的第一个练习**
1. 在终端里，输入以下命令（**一个字一个字敲，不要复制粘贴！**）：
   ```bash
   cd /Users/alalapi/Desktop/computer_study_plan
   ```
   
   > 💡 注意：上面的路径是示例。如果你把项目放在了其他地方，需要改成你的实际路径。

   ```bash
   cd rounds/round_00/week1
   ```

**📈 第5步：查看进度更新**

### 🌟 重要提示（新手必读）

**💪 心态准备**
1. **允许自己犯错**：输错命令、找不到文件都很正常，这是学习过程的一部分

**🆘 常见问题解答**

**Q：我输错命令了怎么办？**
A：按 `Ctrl + C` 中断当前命令，重新输入即可。

**Q：我不知道现在在哪个文件夹？**
A：输入 `pwd` 查看当前位置。

**Q：我想回到桌面？**
A：输入 `cd ~/Desktop` 或 `cd ~/桌面`

**Q：练习脚本运行出错了？**
A：检查你是否在正确的目录，用 `pwd` 确认。

**Q：怎么关闭终端？**
A：输入 `exit` 或直接点窗口的关闭按钮。

### 🆘 遇到困难怎么办？

**1. 卡在某个练习**

- 再看一遍对应的 `notes.md` 文件

- 把错误信息完整复制下来

- 这是正常的！所有人都会忘记
- 查看 `rounds/round_00/final/command_cheatsheet.md` 命令小抄
- 或者重新运行 `exercises.sh` 复习

**4. 感觉进度太慢**
- 这是正常的！Round 00 就是最慢的
- 记住：学习曲线是**先慢后快**的
- 一旦掌握了基础命令，后面的学习速度会指数级增长

## 🤔 新手常见问题解答（FAQ）

### ❓ 基础问题

**Q：我完全不懂编程，能学会吗？**
A：**绝对可以！** 这个计划就是为**零基础**设计的。我们从最基础的“双击打开文件”开始，一步步来。你不需要任何编程经验。

**Q：每天要花多少时间？**
A：建议**每天15-30分钟**，每周3-4天。重要的是**持续**，而不是一次学很久。即使今天只学会了一个命令，也是进步！

**Q：我卡住了怎么办？**
A：
1. **休息5分钟**：喝杯水，站起来走走
2. **再看笔记**：对应的 `notes.md` 文件可能有答案
3. **简化问题**：回到上一步，确认基础操作没问题
4. **搜索错误**：把错误信息复制到搜索引擎
5. **问AI助手**：用简单语言描述你的问题

**Q：我忘了某个命令怎么办？**
A：**这太正常了！** 所有人都记不住所有命令。解决方案：
- 查看 `rounds/round_00/final/command_cheatsheet.md`
- 用 `命令名 --help` 查看帮助
- 用 `man 命令名` 查看详细手册
- 重新运行练习脚本复习

### 💻 技术问题

**Q：在终端里输错了命令怎么办？**
A：
- 按 `Ctrl + C` 中断当前命令
- 重新输入正确的命令
- 或者按 `↑` 键调出上一条命令修改

**Q：不知道当前在哪个文件夹？**
A：输入 `pwd`（Print Working Directory）查看当前位置。

**Q：想回到桌面/家目录？**
A：
- macOS：`cd ~/Desktop` 或 `cd ~/桌面`
- Linux：`cd ~/Desktop`
- 回到用户主目录：`cd ~` 或直接 `cd`

**Q：练习脚本运行出错？**
A：检查：
1. 是否在正确目录：`pwd` 确认
2. 脚本文件是否存在：`ls` 查看
3. 是否有执行权限：`chmod +x exercises.sh`（如果需要）

**Q：进度看板不更新？**
A：
1. 确保脚本正确运行完成
2. 刷新浏览器页面（`⌘R` 或 `F5`）
3. 或者用本地服务器方式（自动刷新）

### 🧠 学习策略

**Q：我看得懂，但写不出来怎么办？**
A：**多动手！** 编程是技能，不是知识。就像学游泳，看再多视频不如跳进水里。每个练习都要**亲手敲**，不要复制粘贴。

**Q：进度太慢了，着急怎么办？**
A：**正常！** 学习曲线是先慢后快的：
- 第1周：学7个命令 → 感觉慢
- 第1个月：掌握基础 → 开始加速  
- 第3个月：形成直觉 → 飞速进步

**Q：学到后面忘了前面的怎么办？**
A：
1. **定期复习**：每周回顾一次命令小抄
2. **实际使用**：学到的命令用在日常工作生活中
3. **做笔记**：在 `command_cheatsheet.md` 添加自己的笔记

### 🎯 心态建议

**记住三件事**：
1. **犯错是学习的一部分**：每个程序员都会写错命令
2. **慢就是快**：基础打牢，后面才能跑得快
3. **完成比完美重要**：先做出能运行的，再慢慢优化

**学习编程就像学外语**：
- 第1天：Hello World（你好）
- 第1周：基本会话（pwd, ls, cd）
- 第1个月：日常交流（文件操作，Python基础）
- 半年后：流畅表达（写完整程序）
