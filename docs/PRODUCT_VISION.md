# Product Vision

> 更新日期：2026-07-18
> 阶段：单课程验证（Linux only）

## 1. 一句话定位

本项目是一个本地优先的网页交互式学习产品原型：当前只服务一门正式课程——**Linux 基础与工程实践**（`linux-foundations`），并用这门课验证游戏化强正反馈学习体验是否成立。

## 2. 产品核心价值

必须同时追求：

- **有趣**：学习过程本身有驱动力，而不是只有任务清单
- **美观**：界面清晰、专注、愿意每天打开
- **真正学到东西**：每个任务对应可验证的 Linux 能力
- **能长期坚持**：降低启动成本，形成可重复的每日循环
- **每个具体任务有记录**：完成、撤销、备注、证据可追溯
- **每次用户动作有反馈**：不是只有“已完成”，而是对动作有回应
- **每个阶段有通关感**：模块可推进、可收口、可复盘
- **奖励必须绑定真实学习行为**：不允许空点按钮刷成就

## 3. 当前产品范围

当前正式课程只有：

**Linux 基础与工程实践**（`linux-foundations`）

覆盖：

- 终端与 Shell 基础
- 文件系统与路径
- 管道、重定向与文本工具
- Bash / Shell 脚本
- 进程、日志、cron
- SSH / SCP / rsync 与远程基础
- Linux 综合练习与项目型实践

明确不在当前正式范围：

- 考研 / 数学二 / 英语 / 政治
- 软考独立课程
- 与 Linux 无关的纯 Python、算法、Web 后端、数据库、机器学习、PyTorch、NLP 等独立课程

## 4. 长期扩展原则

底层可以为多课程保留 course-agnostic 结构，但在满足以下条件之前，**不新增第二门课程**：

1. Linux 单课程完整闭环已经验证
2. 底层 Course / Module / Task / Attempt 模型已经稳定
3. 用户能够完成真实学习任务
4. 反馈和记录机制已经有效
5. UI 体验已经验证
6. 用户明确需要其他课程

## 5. 核心学习循环

```text
选择 Linux 学习任务
→ 阅读或练习
→ 用户执行命令、回答问题或提交成果
→ 系统记录动作
→ 系统验证结果
→ 返回即时反馈
→ 更新掌握度与成长状态
→ 推进关卡
→ 完成阶段挑战
→ 通关并获得成就
→ 推荐下一项 Linux 任务
```

**当前代码尚未完整实现上述循环。**

现已具备的最小闭环：任务展示、内联阅读、练习脚本运行、浏览器终端、记录并完成 / 撤销、动作日志、基础任务反馈、本地存档。

尚未实现：完整 Attempt 模型、规则驱动反馈引擎、XP / Mastery、关卡通关与成就系统、游戏化 UI 重设计。

## 6. 本阶段成功标准

单课程验证成功，至少意味着：

- 用户能持续完成 Linux 任务而不是只逛页面
- 每次关键学习动作都留下记录
- 反馈能帮助下一步，而不是空泛鼓励
- Round 00 到后续 Linux 模块形成清晰通关节奏
- 网页体验让人愿意继续打开
