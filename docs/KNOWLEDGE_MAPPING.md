## 3. 软考与 408 的"同模块不同深度"对照（关键）

> 这张表是**避免重复学习**的核心。
> 同一个模块在软考和 408 里都考，但深度不同；本仓库内一定要分两份笔记 / 错题归类。

| 模块 | 软考深度（广度入门） | 408 深度（计算 + 推理） | 本仓库处理 |
|---|---|---|---|
| 数据结构 | 常见数据结构 + 简单代码 | 综合应用题（含手写算法） | 共享概念笔记，独立错题本 |
| 操作系统 | 概念 + 简单 PV 同步 | 调度算法计算 + 内存管理推导 | 共享概念笔记，独立错题本 |
| 计算机组成 | 数制 + 简单指令系统 | 完整 CPU 流水线 + cache 替换 | **必须独立加深** |
| 计算机网络 | 协议层次 + 功能 | 滑动窗口数值题 + 子网计算 | 共享概念笔记，独立错题本 |
| 数据库 | 关系 / SQL / 范式 | 408 不考 | 仅在软考 |
| 软件工程 | 软考核心 | 408 不考 | 仅在软考 |
| UML / 信息安全 | 软考独有 | 408 不考 | 仅在软考 |

## 4. 数学二与其他线的关系

数学二**与软考、408 几乎不重叠**。它的关键特点：

- 启动时间不能拖
- 长期低强度 > 短期高强度
- 错题积累比看视频更有用
- 与计算机基础不可互替

## 6. 使用本映射表的方法

1. 学习新内容前，先查本表确认它属于哪条 lane / 是否多线共用。
2. 多线共用的模块，笔记在 `plans/<scope>/` 各自建一份；错题统一归类到 `records/error_notes/<lane>/<module>/`。
3. 标记"辅助"的现有 Round md，在 lane 进度看板中**不重复计算**进度，只作为入门素材。
4. 本表每完成一个模块就在"推荐处理方式"列补充实际进展。
5. **不要把数学二的内容塞进软考 / 408 计划**，反之亦然。
