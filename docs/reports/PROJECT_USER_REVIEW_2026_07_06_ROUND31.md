# Project User Review · Round 31

> 日期：2026-07-06
> 视角：普通学习者 + 移动端真实操作
> 范围：Web UI 学习工作区、教程 / 终端联动、移动端滚动、终端快捷命令

## 本轮发现的 5 个问题

1. API 正常时仍显示“网页记录已启用”横幅，占用首屏空间但不提供下一步动作，移动端尤其显得拥挤。
2. 移动端点击“读教程”后，页面仍停在整个学习工作区顶部；教程和终端需要继续滚动寻找。
3. 移动端点击“终端练习”后，页面也没有直接把终端面板顶到视口，用户会觉得终端和教程没有真正联动。
4. 教程面板和终端面板之间没有互跳入口；阅读到一半想去终端，或终端操作后想回教程，都只能手动长滚动。
5. 终端“快捷命令”仍是通用 `pwd / ls` 级别，且“回到 ~/cli-lab”按钮实际通过 `cd ~` 表达，和当前任务沙盒语义不够一致。

## 修复摘要

- 隐藏 API 成功横幅，只保留只读 / 文件模式等需要处理的警告。
- 增加移动端工作区快速切换条：当前任务 / 教程 / 终端。
- `读教程` 在移动端滚到教程面板，`终端练习` 在移动端滚到终端面板并聚焦输入框。
- 在教程面板加入“去终端”，在终端面板加入“回到教程”。
- 将终端快捷命令改为任务上下文命令，按当前 week 增加 `mkdir -p weekN/self_check` 和 `cd weekN/self_check`；终端重置改为直接回到 `~/cli-lab` 并显示系统提示。

## 验证结果

- `node --check progress_ui.js` 通过。
- `python3 scripts/agent_gate.py --verify` 通过。
- 浏览器移动端 Round 02 回检通过：成功横幅隐藏；快速切换条显示“当前任务 / 教程 / 终端”；快捷命令包含 `week1/self_check`；点击“读教程”后教程面板进入视口；点击“终端练习”后终端面板进入视口且输入框聚焦；终端重置后显示 `~/cli-lab`。
- 浏览器回检默认桌面、Round 02 桌面和 Round 02 移动端均可加载，页面显示“学习工作区 / 读教程 / 记录并完成”，且无横向溢出。
