# Web UI 用户视角第二轮评测报告（2026-07-06）

## 评测范围

在 `TASK-RR-54` 第一轮改版后，继续从用户视角检查首屏效率、移动端可见性和任务按钮语义。

## 累计发现的 5 个新问题

1. **顶部说明仍偏高**：`今日学习` 大标题和说明占据较多垂直空间，学习工作区仍被推到较低位置。
2. **API 状态横幅过重**：成功状态横幅是整块提示，视觉权重接近主任务区，会抢当前任务的注意力。
3. **移动端导航末项被裁切**：390px 视口下侧边导航横向滚动，`进度设置` 首屏只露出一部分，入口不够明确。
4. **脚本任务按钮语义不准**：`.sh` / `.py` 练习任务仍显示“读教程”，但打开的是脚本内容，用户会误解。
5. **终端绑定滚动位置不稳定**：从 Round 清单点击“终端练习”后，浏览器可能把焦点滚到输入框附近，学习工作区标题和任务顶部不稳定。

## 本轮修复方向

- 压缩顶部 `今日学习` 信息密度。
- 降低 API 成功横幅的视觉权重。
- 移动端导航改为自然换行，不依赖横向滚动。
- 根据文件类型显示“读教程 / 看脚本 / 打开资料”。
- 终端绑定后使用确定性滚动，并避免输入框聚焦触发额外滚动。
