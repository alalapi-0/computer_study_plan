# Web UI 用户视角评测报告（2026-07-06）

## 评测范围

- `README.md` 是否能让新用户明确启动和使用方式。
- `progress.html` 桌面端用户路径是否清晰。
- Round 深链、教程阅读、浏览器终端、任务记录是否能形成一条自然流程。

## 累计发现的 5 个问题

1. **首屏信息过载**：今日学习、总进度、四主线、本周配置、终端、存档、阶段、薄弱项、Round 清单全部纵向堆叠，用户很难判断现在应该先做什么。
2. **教程和终端割裂**：教程默认用居中的阅读弹窗打开，会遮住页面和终端；用户必须关闭教程才能敲命令。
3. **指引不够明确**：页面文案说能闭环，但没有把“读教程 -> 敲命令 -> 写记录 -> 完成”组织成首屏可见的操作步骤。
4. **Round 深链与首屏任务不一致**：访问 `progress.html?round=round_02` 时，底部 Round 02 被选中，但首屏“下一步任务”仍指向软考计划入口。
5. **任务行动入口重复**：同一任务行同时有“学习资料：notes.md”和“打开”两个阅读入口，用户需要猜哪个才是主操作。

## 本轮修复方向

- 将“当前任务、教程、浏览器终端”合并为首屏学习工作区。
- 教程默认在页面内联阅读，不遮挡终端；弹窗阅读变成可选。
- 非核心管理信息折叠到“进度、配置、存档与复盘信息”。
- `?round=round_XX` 和用户点击 Round 后，首屏任务同步切换到对应 Round。
- 任务行只保留一个明确的“读教程”入口，命令类任务保留“终端练习”入口。

- `npm run check:cursor-mcp` 显示 Cursor CLI 层多数 MCP 为 `needs approval`，但可列出 chrome-devtools / playwright 工具清单；本轮使用 Codex 内嵌浏览器控制进行真实页面 before 检查。
- 内嵌浏览器 DOM 快照接口报兼容错误，改用截图和只读页面评估 API 完成验证。
