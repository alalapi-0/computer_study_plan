# Project User Review · Round 27

> 日期：2026-07-06
> 视角：普通学习者 + 后续自动推进 Agent
> 范围：Web UI 可见品牌、静态模式提示、Round 00 入口、转换协议、总路线与知识映射

## 本轮发现的 5 个问题

1. Web UI 左侧品牌标题仍显示“学习进度工作台”，和页面标题、主流程里的“学习工作区”不一致。
2. `progress_ui.js` 的本地文件模式提示仍说“访问看板”，用户直接双击页面时会继续看到旧定位。
3. `rounds/round_00/README.md` 的进度追踪入口仍写“打开看板”，和当前教程 / 终端 / 记录同屏的学习工作区不一致。
4. `CONVERSION_PROTOCOL.md` 的打开方式章节和变更记录仍使用“进度看板 / 看板打开方式”，会误导后续新增 Round 时继续按旧口径写文档。
5. 总路线和知识映射仍有事实口径残留：`MASTER_STUDY_ROADMAP` 把映射表写成“20 模块”，`KNOWLEDGE_MAPPING` 仍把 lane 视图叫“进度看板”。

## 修复摘要

- 将 Web UI 左侧品牌标题改为“学习工作区”。
- 将本地文件模式提示改为建议通过本地服务访问 Web UI 学习工作区。
- 将 Round 00 进度追踪入口改为打开 Web UI 学习工作区。
- 将转换协议的打开方式章节和变更记录从看板口径改为 Web UI 学习工作区。
- 将知识映射总数修正为 24 模块，并把 lane 展示描述改为 Web UI lane 视图。

## 验证结果

- 文本扫描确认本轮目标旧说法不再残留。
- 浏览器回检默认 Web UI、桌面 Round 02 与移动端 Round 02：页面标题为“学习工作区 · 四主线”，左侧品牌标题为“学习工作区”，教程入口、`记录并完成`、Round 02 终端和 `~/cli-lab/round2` 绑定均正常，无横向溢出。
- `python3 scripts/agent_gate.py --verify` 通过。
