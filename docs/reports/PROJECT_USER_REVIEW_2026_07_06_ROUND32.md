# Project User Review · Round 32

> 日期：2026-07-06
> 视角：普通学习者 + 非终端任务移动端操作
> 范围：Web UI 学习工作区、非工程任务入口、记录弹窗、动作记录命名

## 本轮发现的 5 个问题

1. 默认软考阅读任务不需要终端，但移动端快速切换条仍显示“终端”，点击后会跳向一个隐藏面板。
2. 非终端任务的教程面板仍显示“去终端”，和当前任务“只需阅读并记录”的状态冲突。
3. 移动端仍显示“单独打开教程”按钮，容易把用户带回遮挡工作区的弹窗阅读模式。
4. 记录弹窗把可写示例只放在 placeholder 里，用户打开弹窗时看不到“应该写什么”的具体参考。
5. 动作记录里 `run_exercise` 显示为“运行练习”，但页面主按钮叫“运行脚本”，命名不一致。

## 修复摘要

- 非终端任务下隐藏移动端快速切换条中的“终端”入口。
- 非终端任务下隐藏教程面板里的“去终端”入口。
- 移动端隐藏“单独打开教程”按钮，优先保留内联教程。
- 记录弹窗增加可见的“记录参考 / 证据示例”块。
- 将动作记录中的 `run_exercise` 标签统一为“运行脚本”。

## 验证结果

- `node --check progress_ui.js` 通过。
- `python3 scripts/agent_gate.py --verify` 通过。
- 浏览器默认移动端回检通过：软考任务处于 no-terminal 状态，快速切换条只显示“当前任务 / 教程”，教程面板不再显示“去终端”，终端面板隐藏，且无横向溢出。
- 浏览器 Round 02 移动端回检通过：工程任务仍显示“当前任务 / 教程 / 终端”，教程面板仍显示“去终端”，终端面板可见。
- 浏览器默认桌面回检通过：软考任务无终端入口，页面显示“学习工作区 / 记录并完成”，且无横向溢出。
- 记录弹窗回检通过：软考任务和 Round 02 工程任务均显示“记录参考 / 证据示例”。
