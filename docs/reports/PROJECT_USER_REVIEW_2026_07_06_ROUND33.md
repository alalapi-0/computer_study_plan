# Project User Review · Round 33

> 日期：2026-07-06
> 视角：普通学习者 + 记录弹窗填写
> 范围：任务反馈生成、记录弹窗示例、教程弹窗按钮、反馈 JSON

## 本轮发现的 5 个问题

1. 任务反馈生成器对所有未开始任务都使用同一句“先阅读资料，或完成一条最小练习”，没有区分软考、数学二、408 和工程实操。
2. 默认软考任务的当前任务提示因此仍像工程练习提示，不能把用户引向“易混点 / 概念地图”这类软考产物。
3. 记录弹窗直接复用了 placeholder 文案，导致可见示例里出现“例如”重复，尤其是“证据示例：例如：...”。
4. 记录弹窗里的证据路径没有设置强制换行，长路径在窄屏或长目录名下有溢出风险。
5. 桌面端教程弹窗按钮仍叫“单独打开教程”，没有明确说明它会打开遮挡工作区的弹窗。

## 修复摘要

- 将 `scripts/progress_lib.py` 的未开始任务建议改为按 lane 生成：软考指向易混点 / 概念地图，数学二指向例题 / 易错点，408 指向推导点 / 软考差异，工程实操指向终端练习。
- 重新生成 `records/feedback/task_feedback.json`。
- 记录弹窗可见示例去掉 `例如：` 前缀，避免“证据示例：例如：...”。
- 为记录弹窗证据路径增加 `overflow-wrap: anywhere`。
- 将桌面端教程弹窗按钮改为“弹窗查看”。

## 验证结果

- `node --check progress_ui.js` 通过。
- `python3 -m json.tool records/feedback/task_feedback.json` 通过。
- `python3 scripts/agent_gate.py --verify` 通过。
- 抽样确认 `soft_exam`、`math2`、`cs408`、`engineering` 的未开始任务建议已按 lane 区分。
- 浏览器默认桌面 / 默认移动端 / Round 02 移动端回检通过：当前任务提示变为 lane 对应建议，记录弹窗不再出现“证据示例：例如”，移动端无横向溢出。
