# Error Review System · 错题与复盘系统（v1.0）

> 本文档统一定义软考、数学二、408 三条考试导向 lane 的错题与复盘系统。
> 错题不是"记下来就行"，而是要回流到学习计划和优先级。
> 一切错题文件归档到 `records/error_notes/<lane>/<module>/`，周复盘归档到 `records/weekly_reviews/`。

---

## 2. 错题文件命名

```
records/error_notes/<lane>/<module>/YYYY-MM-DD-shortid.md
```

- `YYYY-MM-DD`：你记下这道题的日期（不一定是做这道题的日期）
- `shortid`：3–8 字母数字，本机可辨识即可（不要求全局唯一）

示例：

- `records/error_notes/soft_exam/os/2026-05-12-pv-q3.md`
- `records/error_notes/math2/limits/2026-05-12-lim-2x-1.md`
- `records/error_notes/cs408/network/2026-05-12-sliding-window.md`

## 3. 错题记录模板

每个错题文件应包含以下字段（**强烈建议**），但不要求完全填满。能写多少写多少：

```markdown
# 错题：[一句话自写题意摘要]

## 元信息

| 字段 | 内容 |
|---|---|
| 日期 | YYYY-MM-DD |
| 来源 | （如：软考 20XX 上半年真题 / 数学二 20XX 真题 / 模拟题 / 教材习题 / 自创） |
| 题目类型 | 选择 / 填空 / 简答 / 计算 / 综合 / 编程 |
| 所属 lane | soft_exam / math2 / cs408 |
| 所属模块 | （对照 §1 的子目录） |
| 难度 | 低 / 中 / 高 |
| 上次复习日期 | YYYY-MM-DD 或 — |

## 题意摘要与来源定位（必填）

[用自己的话写 1-3 句题意摘要，不复制完整题面。写清楚来源、页码、题号、截图在本机的位置或教材定位，方便回到原资料复习。]

## 我的错误答案

[写自己当时的答案或思路摘要，不需要复制原题完整内容。]

## 正确答案

[写关键结论、推导步骤或订正思路。若答案来自教材/官方解析，只写定位和自己的理解，不整段复制。]

## 错误原因（必选 1+）

- [ ] 概念不清：______
- [ ] 计算错误：______
- [ ] 审题错误：______
- [ ] 公式/定理记错：______
- [ ] 思路偏差：______
- [ ] 时间不够：______
- [ ] 其他：______

## 正确知识点

- 涉及概念：______
- 关键公式/定理：______
- 解题套路：______

## 是否需要回炉学习？

- [ ] 是 → 回到哪份笔记？`plans/<lane>/<module>.md`
- [ ] 否（只是马虎）

## 下次复习计划

- 第一次复习：YYYY-MM-DD（建议 3 天后）
- 第二次复习：YYYY-MM-DD（建议 7 天后）
- 第三次复习：YYYY-MM-DD（建议 30 天后）

## 错题次数计数

- 已错次数：1
- （下次再错回来这份文件把数字 +1）
```

> 不要求模板"看起来完美"。**有题意摘要 + 错误原因 + 订正思路**这 3 项就比没有强 100 倍。

## 5. 错题回流机制（关键）

> 错题不回流 = 错题白记。本节是整个系统真正发挥作用的部分。

### 5.1 触发条件

| 信号 | 触发动作 |
|---|---|
| 同一道题连续错 **2 次** | 加入"薄弱项"列表（写在本周复盘） |
| 同一道题连续错 **3 次** | 强制回炉对应章节笔记 `plans/<lane>/<module>.md` |
| 高频错题（**4 周内同模块 ≥ 5 道**） | 进入周复盘"高频错题"段；下周计划主推该模块 |
| 高频模块（**8 周内同模块 ≥ 10 道**） | 在 `docs/KNOWLEDGE_MAPPING.md` 把该模块优先级升到 P0 |

### 5.2 回流落到哪些文件

| 动作 | 落地位置 |
|---|---|
| 加入薄弱项 | `records/weekly_reviews/YYYY-WW.md` 中"高频错题模块"段 |
| 强制回炉章节 | `plans/<lane>/<module>.md` 文件头部加一行"⚠ YYYY-MM-DD 错题驱动回炉" |
| 下周计划主推该模块 | 写在 `records/weekly_reviews/YYYY-WW.md` 的"下周计划"段 |
| 调整 KNOWLEDGE_MAPPING 优先级 | 修改 `docs/KNOWLEDGE_MAPPING.md` 主表的"优先级"列 |

### 5.3 错题计数器

每份错题文件末尾保留"已错次数"。**每次再错回到该文件 +1**，而不是新建一份。文件命名以**第一次错的日期**为准。

## 6. 每周错题复盘流程

每个周复盘必须包含以下错题相关内容（与 `docs/WEEKLY_EXECUTION_TEMPLATE.md` 配合）：

```markdown
## 本周错题（必填）

- 新增数量：__ 道
- 按 lane 分布：
  - soft_exam: __
  - math2: __
  - cs408: __
- 高频模块（≥ 3 道）：______
- 强制回炉触发：[ ] 是（章节：______） / [ ] 否

## 本周错题回流动作

- [ ] 已在 plans/<lane>/<module>.md 标注回炉
- [ ] 已在下周计划主推该模块
- [ ] 已调整 KNOWLEDGE_MAPPING 优先级
```

## 7. 模考结果记录（仅软考 / 408 适用）

模考与错题分开记录，但**最终错题归并到错题本**。

模考文件位置：

```
records/weekly_reviews/mock_exams/YYYY-MM-DD-<lane>-<name>.md
```

模考文件应包含：

- 模考日期、来源、用时、总分、各模块得分
- 弱项模块识别
- 弱项模块下"具体错题清单"链接到 `records/error_notes/...`

---

## 8. 错题与 progress.json 的关系（重要）

- 错题文件**不进入** `progress.json`。
- `progress.json` 只跟踪"我注册的、显式的任务"是否完成。
- 错题数量、错题次数、错题回流都通过文件系统 + 周复盘文档表达。
- 这个设计的好处：JSON 简单，可校验，不污染；错题层有版本控制，且可读。

---

## 9. 实操步骤示例

1. 做题时遇到错题 → 暂时在草稿本上做。
2. 当天或次日 → 把错题录入 `records/error_notes/<lane>/<module>/YYYY-MM-DD-shortid.md`。
3. 每周复盘 → 统计新增数量 + 高频模块。
4. 触发回流 → 标注章节回炉 / 调整下周计划 / 调整 KNOWLEDGE_MAPPING。
5. 复习日 → 打开错题文件，看一遍，"已错次数"按需 +1。

---

## 10. 不要做的事

- 不要把错题文件改名："2026-05-12-xxx.md" 改成 "important-xxx.md" 后无法按时间排序。
- 不要把错题塞进 `plans/<lane>/<module>.md`：plans 是"课件骨架 + 笔记"，错题在 records。
- 不要"批量复习"完全跳过"已错次数"更新：复习次数是衡量是否真的掌握的关键指标。
- 不要在错题本里写情绪话："这题太蠢了，我居然错了！"——错题本是工程文档，情绪写在周复盘。
