# VPS 安全 Checklist

> 本清单适用于所有涉及 VPS 操作的 Round。
> 编程 AI 在执行 Level 2 及以上远程操作前，必须能逐项打勾。

## 1. 凭证与敏感信息

- [ ] 仓库内**没有**真实服务器 IP。
- [ ] 仓库内**没有**真实用户名。
- [ ] 仓库内**没有**真实密码。
- [ ] 仓库内**没有** SSH 私钥（`id_rsa` / `id_ed25519` 或同类文件）。
- [ ] 仓库内**没有**真实 API Key / token。
- [ ] 真实 `.env` 文件**未**被 Git 跟踪。
- [ ] 仓库存在 `.env.example` 但其中不含真实凭证。
- [ ] `.gitignore` 已包含 `.env`、`.venv`、`*.pem`、`id_rsa*`。

## 2. SSH 与登录

- [ ] 优先使用 SSH 密钥登录，而非密码登录。
- [ ] 私钥保存在本地 `~/.ssh/`，权限 `600`。
- [ ] 未将私钥同步到任何远程仓库或云盘。
- [ ] 已知服务器的 SSH 端口（默认 22 或自定义端口）。
- [ ] 已记录如何回滚 `sshd_config` 的改动方案（如果本轮涉及）。

## 3. 命令安全

- [ ] 所有命令在执行前已经过通读，确认含义。
- [ ] 没有出现来自不可信来源的一键脚本（`curl ... | bash`）。
- [ ] 没有未确认的 `rm -rf`、`mkfs`、`dd`、`shutdown`、`reboot`。
- [ ] 没有未确认的 `chmod 777` 或 `chown` 大范围操作。
- [ ] 高风险命令在终端执行前再次人工核对。

## 4. 端口与防火墙

- [ ] 没有在未授权情况下开放新的对外端口。
- [ ] 服务默认绑定 `127.0.0.1`，公网开放需进入 Level 4 并经用户确认。
- [ ] 未关闭防火墙；如调整规则，已记录回滚方法。
- [ ] 已了解当前服务器使用的是 `ufw` / `firewalld` / `iptables` / 云厂商安全组中的哪一种。

## 5. 服务运行

- [ ] 长任务使用 tmux 或 systemd 管理（systemd 仅 Level 5 授权后使用）。
- [ ] 已记录服务的启动命令、停止命令、回滚命令。
- [ ] 已为服务输出文件日志，知道日志路径。
- [ ] 未默认配置开机自启（除非 Level 5 已授权）。

## 6. 备份与回滚

- [ ] 涉及修改 / 删除的关键目录已确认有备份或可重新生成。
- [ ] 涉及配置变更的文件（如 `sshd_config`、`nginx.conf`）已先复制为 `.bak`。
- [ ] 已写好回滚步骤；不依赖记忆。

## 7. 操作记录

- [ ] 本轮全部命令、时间、结果已写入对应 Round 的输出物。
- [ ] 错误码与异常输出已被记录（用于后续排查）。
- [ ] 操作记录中不含真实 IP / Key / 私钥（用占位符替换）。

## 8. 高风险操作（Level 5）专用

执行任何 Level 5 操作前还必须：

- [ ] 已说明操作目的。
- [ ] 已列出完整命令清单。
- [ ] 已说明影响范围。
- [ ] 已说明回滚方案。
- [ ] 已取得用户**逐项**明确确认。
- [ ] 操作后立即写入操作日志。

> 任何一项未勾选，都视为本轮远程操作**未通过安全检查**，必须停止。
