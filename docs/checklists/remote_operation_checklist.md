# 远程操作执行 Checklist

> 本清单是 Level 2 / 3 / 4 / 5 远程操作的**统一执行检查表**。
> 每个涉及真实服务器的 Round 都应在执行前完成本清单。

## 阶段 A · 执行前

- [ ] 已确认本轮目标，且目标可以一句话说清。
- [ ] 已判定本轮所需的最高权限等级（Level 2 / 3 / 4 / 5）。
- [ ] 已使用 `docs/templates/remote_operation_confirmation.md` 提交确认请求。
- [ ] 已取得用户明确授权（书面 / 对话记录）。
- [ ] 已通读 `docs/checklists/vps_safety_checklist.md` 并逐项检查。
- [ ] 已准备好回滚方案。
- [ ] 已确认本地终端与远程终端的区分（不会"以为在远程其实在本地"）。

## 阶段 B · 连接 VPS

- [ ] 使用 SSH 密钥登录，而非密码。
- [ ] 登录后第一条命令是 `whoami` / `pwd` / `hostname` 之一，先确认身份与位置。
- [ ] 不在登录后立刻执行写入操作，先做一次只读检查。

## 阶段 C · 执行命令

- [ ] 每条命令执行前再次复读。
- [ ] 高风险命令（`rm`、`chmod`、`chown`、`systemctl`、防火墙）逐条人工确认。
- [ ] 不批量粘贴未审阅的命令块。
- [ ] 任务进度每完成一步，简短记录一次。

## 阶段 D · 服务运行（仅 Level 4）

- [ ] 服务先绑定到 `127.0.0.1` 自测。
- [ ] 用 `curl http://127.0.0.1:<port>` 验证。
- [ ] 服务通过 tmux 或 systemd（仅 Level 5 授权后）管理。
- [ ] 日志已落盘，知道路径。
- [ ] 公网访问需求已被单独确认。

## 阶段 E · 退出

- [ ] 已确认临时进程已停止（除非本轮目的就是后台运行）。
- [ ] 已确认未遗留对外开放的端口。
- [ ] 已确认未遗留临时凭证（如 `.env` 测试值）。
- [ ] 已确认 tmux 会话状态（保留或关闭）。
- [ ] 已 `exit` 退出 SSH 会话。

## 阶段 F · 操作记录

- [ ] 本轮命令清单、时间、结果均已记录。
- [ ] 操作记录中**未**出现真实 IP / Key / 用户名。
- [ ] 操作记录已附在 Round 输出物中。
- [ ] 涉及失败时，错误码与原因已被记录。

## 阶段 G · 后续

- [ ] 是否需要在 `docs/PROJECT_STATE.md` 中更新阶段状态？
- [ ] 是否需要在 `docs/NEXT_ACTIONS.md` 中追加后续任务？
- [ ] 是否需要在 `docs/DECISIONS.md` 中记录新决定？
- [ ] 是否需要在 `docs/checklists/vps_safety_checklist.md` 中补充新发现的安全条款？
