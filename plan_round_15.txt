第 1 周：路径参数和查询参数

这一周只做一件事：把 读接口 写稳。

FastAPI 明确区分了路径参数和查询参数：路径参数是 URL 路径的一部分，查询参数通常出现在 ? 后面；非路径参数的函数参数会自动被当成 query parameters。它们都会按你声明的 Python 类型做解析和校验。 

练习 1：先把 GET /runs/{run_id} 写成真正的路径参数接口

在 api.py 里保留或补成这样：

from fastapi import FastAPI, HTTPException
from db import init_db, get_run_by_id

app = FastAPI()

@app.get("/runs/{run_id}")
async def run_detail(run_id: int):
    init_db()
    row = get_run_by_id(run_id)
    if row is None:
        raise HTTPException(status_code=404, detail="run not found")
    return {"run": row}

这里最重要的不是代码量，而是你要真正看到：
如果 run_id 声明成 int，FastAPI 会按整数处理它，并在非法输入时给出校验错误。路径参数教程就是围绕这个能力展开的。 

练习 2：给 GET /runs 加最小分页参数

把 GET /runs 改成类似这样：

from db import init_db, list_runs

@app.get("/runs")
async def runs(skip: int = 0, limit: int = 10):
    init_db()
    rows = list_runs()
    return {"runs": rows[skip: skip + limit]}

然后在浏览器或 /docs 里测试：
	•	/runs
	•	/runs?skip=0&limit=2
	•	/runs?skip=2&limit=2

FastAPI 查询参数教程明确说：你可以同时声明路径参数和查询参数，而且不需要特意按顺序写；查询参数会按名字识别并自动转换类型。 

练习 3：给 GET /runs 再加一个筛选参数

继续改：

@app.get("/runs")
async def runs(skip: int = 0, limit: int = 10, keyword: str | None = None):
    init_db()
    rows = list_runs()

    if keyword:
        rows = [r for r in rows if keyword in str(r)]

    return {"runs": rows[skip: skip + limit]}

你这一题是在练一个很重要的直觉：
	•	路径参数更像“我想要哪个资源”
	•	查询参数更像“我想怎样筛、怎样分页、怎样控制结果”

这和 FastAPI 对 path / query 的分工完全一致。 

练习 4：观察非法输入

请你故意在 /docs 里或地址栏里试一次错误输入，比如：
	•	GET /runs/foo

你要亲自看一次 FastAPI 自动返回的验证错误。FastAPI 的路径参数和查询参数教程都强调了这一点：类型声明不仅是编辑器提示，也是运行时解析和验证的一部分。
