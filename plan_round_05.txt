第 1 周：双指针、滑动窗口、二分查找

这一周专门练“线性序列上的高频模式”。

你这周要学到什么程度

你不需要背模板，只需要先学会认问题：
	•	两端往中间夹，常常是双指针
	•	固定长度或可伸缩区间，常常是滑动窗口
	•	有序数组里找值、边界、插入位置，常常是二分

CSES Handbook 对 two pointers 的定义非常适合初学者：两个指针都只朝一个方向移动，因此整体仍是线性时间；它还用子数组和、2SUM 做了典型例子。Hello Algo 的二分章节则强调：二分建立在数据有序基础上，每轮把搜索区间减半；10.2 和 10.3 还专门讲了插入位置和边界问题。 

这周的学习顺序

先学双指针，再学滑动窗口，最后学二分。
原因很简单：双指针和滑动窗口都建立在“区间移动”的直觉上，二分则是“利用有序性砍半”。

这周练什么

先做这 4 个小练习：
	1.	Two Sum（排序后双指针版）
	2.	固定窗口求最大和
	3.	最长不重复子串或最长满足条件子数组
	4.	二分查找目标值 / 插入位置

CSES Handbook 已经给了 two pointers 的两个最经典例子：子数组和与 2SUM；Hello Algo 的 binary search / insertion point / edge cases 刚好对应后二者。 

这周的微型成果

做一个 sequence_patterns.py，里面放 3 个函数：
	•	two_sum_sorted(nums, target)
	•	max_sum_subarray_k(nums, k)
	•	binary_search_insert(nums, target)

这一周结束时，你要有这种感觉：

“数组题不是都靠暴力，很多时候只是窗口怎么动、边界怎么收。”

第 2 周：排序、分治、DFS / BFS、回溯

这一周开始从“线性序列”切到“递归和遍历”。

你这周要学到什么程度

排序这块，你先不追求把所有排序都背全。你只抓住 3 个：
	•	插入排序：帮助你理解局部有序
	•	归并排序：帮助你理解分治
	•	快速排序：帮助你理解 partition 思想

VisuAlgo 的 sorting 页面明确说排序是把数组或列表重新排列成某种次序，并支持多种排序算法的动画；Hello Algo 的 divide and conquer 章节则直接用 merge sort 讲“分而治之”：先拆成小问题，再把子问题结果合并回来。 

DFS / BFS 这周怎么学

这一周你只要形成一个很稳的直觉：
	•	DFS 更像“先一路走到底，再回退”
	•	BFS 更像“一层一层扩散”

VisuAlgo 的 graph traversal 页面明确写到，DFS 和 BFS 在图遍历里都能以 O(V+E) 运行，并展示了 reachability、connected components、topological sort、bipartite checking 等典型用途；同站的单源最短路页面还特别说明：在无权图里，BFS 可以解决最少边数意义下的最短路。 

回溯这周怎么学

这一周你只需要会识别两件事：
	•	选择空间很大
	•	需要“试一个选择，不行就撤回”

Hello Algo 对回溯的定义非常直接：它本质上是一种穷举搜索方法，从初始状态出发，不断尝试所有可能选择；总结页又进一步强调，它是对解空间做深度优先遍历。 

这周练什么

先做这 4 类练习：
	1.	归并排序的手动模拟
	2.	二维网格 DFS / BFS
	3.	简单迷宫最短步数（BFS）
	4.	全排列或子集生成（回溯）

VisuAlgo 的 recursion 页面很适合这周一起用，因为它能把递归树和 DP 的 DAG 对比出来；这对你理解“为什么有些递归会重复算”特别有帮助。 

这周的微型成果

做一个 traversal_lab.py，至少包含两个功能：
	•	给一个 2D 网格，找从起点到终点是否可达
	•	给一个小列表，输出所有排列或所有子集

这一周结束时，你要有这种感觉：

“DFS / BFS / 回溯并不是三套毫无关系的东西，它们本质上都在做状态空间搜索，只是扩展顺序不同。”

第 3 周：贪心、DP 入门、模式区分

这一周是“收口周”。不是让你深挖 DP，而是先学会区分：

这题更像贪心，还是更像 DP。

贪心要学到什么程度

Hello Algo 的 greedy 章节和总结都强调：贪心与 DP 都常用于优化问题，但贪心每一步只做当前最优选择；它通常实现简单、效率高，但并不总能保证全局最优，经典反例之一就是某些币值体系下的换硬币问题。 

所以这周你只要记住：
	•	局部最优很自然时，先试贪心
	•	但别把“看起来合理”误当成“必然最优”

DP 先学到什么程度

Hello Algo 对 DP 的引言和总结非常适合你这个阶段：DP 通过把大问题拆成子问题，并保存已经算过的子问题答案，来避免重复计算；它与回溯的关系也讲得很清楚——很多 DP 问题如果不考虑效率，能先从回溯出发，再发现有大量重叠子问题，最后才转成 DP。 

你这周只学两个入门例子就够：
	•	爬楼梯
	•	最小路径和 或 打家劫舍 这一类

这周最重要的能力

不是会写很多 DP，而是先会这 3 步：
	1.	看递归 / 暴力解里有没有重复子问题
	2.	能不能定义“状态”
	3.	当前答案是不是由更小状态转移过来

Hello Algo 的 DP solution pipeline 页面就把这件事拆成了很清楚的步骤：先想每一步决策，定义状态，再推出状态转移。 

这周练什么

只做这 4 个就够：
	1.	找零钱（先试贪心，再看为什么不一定对）
	2.	活动选择 / 区间调度的简单版贪心
	3.	爬楼梯（DP 入门）
	4.	从回溯版爬楼梯改到记忆化搜索或表格 DP

这周的微型成果

做一个 optimization_lab.py，包含：
	•	一个贪心例子
	•	一个 DP 入门例子
	•	一段注释，写清楚“为什么这个更像贪心 / 为什么这个必须保存子问题结果”

这一周结束时，你要有这种感觉：

“DP 不是玄学，它往往只是把会重复算的递归，变成不重复算。”
