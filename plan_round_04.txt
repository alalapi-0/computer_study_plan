第 1 周：数组、字符串、列表思维

这一周只抓一个核心：顺序存储。

Hello Algo 对数组的定义是：数组是在连续内存空间中存储同类型元素的线性结构；它的优势是随机访问。Hello Algo 还专门总结了数组和链表的差异：数组支持随机访问、内存占用更省，但插入和删除不高效，且初始化后长度不可变。 

这一周你需要形成的直觉是：
	•	顺序扫一遍列表很自然
	•	按索引取某个位置很自然
	•	在中间频繁插入删除就不那么自然
	•	字符串很多时候也可以按“顺序结构”来思考。Python 官方教程把数据结构章节的入口直接放在 list 上，说明对 Python 学习者来说，列表就是最基础的容器起点。 

这周建议你做 3 类练习：
	1.	顺序遍历列表，求和、找最大值、统计满足条件的元素个数
	2.	处理字符串，做字符计数、去空格、简单分词
	3.	比较“追加到末尾”和“插到最前面”的使用感。Python 官方日文教程明确提到：列表末尾 append/pop 很快，但头部 insert/pop 会慢，因为后面的元素都要移动。 

如果你想看动画，这周可以配合 VisuAlgo 的 array 页面和 sorting 页面，用来直观看“顺序存储”和“元素移动”。 

第 2 周：链表、栈、队列

这一周重点是把“顺序存储”切换成“节点连接”的脑子。

Hello Algo 对链表的解释很适合初学者：链表的元素是节点，节点之间通过引用连接；它的优势在于不要求像数组那样占用一整块连续内存。Hello Algo 还特别指出，单链表常被用来实现栈、队列、哈希表和图。 

你这一周不用沉迷手写各种花式链表实现。真正要抓住的是：
	•	为什么链表插入删除可以更灵活
	•	为什么它不擅长按下标随便跳着访问
	•	为什么栈和队列都可以建立在数组或链表上。Hello Algo 总结里明确说：栈遵循 LIFO，可以用数组或链表实现；队列遵循 FIFO，也可以用数组或链表实现。 

这周建议的理解顺序是：

先看链表动画，再看栈，再看队列。
因为链表是“结构思维”，栈和队列是“受限操作规则”。Hello Algo 对栈的定义是 LIFO，对队列的定义是 FIFO，这一层你必须非常清楚。 

Python 里的实践建议也很明确：
	•	栈：先直接用 list.append() / list.pop() 理解
	•	队列：优先用 collections.deque，不要养成 list.pop(0) 的习惯。Python 官方教程和 collections 文档都给了这个建议。 

这周建议做 3 类练习：
	1.	用列表模拟“后进先出”的撤销记录
	2.	用 deque 模拟任务排队
	3.	自己画出一个 3 到 5 个节点的单链表，把“插入一个节点”和“删除一个节点”画清楚。VisuAlgo 的 list 页面很适合你做这件事，因为它把 singly / doubly linked list、stack、queue、deque 放在同一套可视化里。

counter = {}

你这一轮最容易踩的坑

第一个坑，是把“背定义”当成“会用了”。
真正重要的是：你看到场景，能选结构。比如高频头部出队，就不要本能地写 list.pop(0)。Python 官方教程已经明确提醒了这一点。 

第二个坑，是一上来就沉迷手写底层实现。
你现在的目标不是面试表演，而是先建立工程里的结构选择感。所以这轮先把 Python 内置容器和 deque 用顺。Python 官方文档已经给了很清楚的建议路径。 

第三个坑，是把哈希表理解成“万能快表”。
Hello Algo 也专门把 hash collision 单独列出来，这就说明哈希并不是魔法，只是非常高频、非常实用的工具。
