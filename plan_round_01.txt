Round 1 的定位

这轮不是学 Linux 全家桶，也不是学 shell 编程。它只解决一件事：

让你在命令行里不慌，能自己找文件、建目录、看文件、移动文件、删测试文件、查帮助。

Apple 官方把 Terminal 定义为 macOS 里用命令行执行任务的应用；Missing Semester 把 shell 解释为一种文本接口，核心能力是运行程序、给程序输入、查看输出。把这两个概念先分清，你后面学 Shell、Git、Python 才不会混。 

这轮学什么，不学什么

这轮只学这些命令：

pwd、ls、cd、mkdir、touch、cp、mv、rm、cat、less、head、tail、man

这轮先不学 chmod、sudo、Git、管道、Shell 脚本。Ubuntu 官方教程虽然也会提到 superuser 和 command chaining，但对你现在的目标来说，Round 1 先把“文件系统与基础命令”打牢，收益更高。

Ubuntu 教程明确把 “Creating folders and files” 和 “Moving and manipulating files” 作为核心章节；它还特别提醒了命令行文件名大小写、转义和命名习惯这些细节。

这轮的唯一小项目

做一个最小目录实验室：

mkdir -p ~/cli-lab/round1/study_files_lab
cd ~/cli-lab/round1/study_files_lab
mkdir input output backup notes
touch input/raw1.txt input/raw2.txt input/raw3.txt
cp input/raw1.txt backup/
mv input/raw2.txt output/
mv input/raw3.txt input/final_raw3.txt
touch notes/round1_notes.txt

然后用 cat 或 less 打开 notes/round1_notes.txt，自己写下每个命令的中文理解。

这个小项目的价值在于，它已经开始模拟你后面做 AI 项目时最常见的目录组织方式：input 放原始数据，output 放结果，backup 放备份，notes 留说明。命令行之所以长期有用，Ubuntu 官方给出的原因也很直接：它在跟着在线说明操作、管理文件、把多个命令串起来时，依然有明显的灵活性和效率优势。

第 1 周：路径感训练

这一周只练 3 个核心动作：
	•	我现在在哪
	•	这里有什么
	•	我怎么切换到别的目录

Apple 和 Ubuntu 的入门材料都把这类基础导航放在最前面，重点就是先熟悉 pwd、ls、cd。 

练习 1：认识当前位置

cd ~/cli-lab/round1
pwd
ls

你要知道：
	•	pwd：显示当前路径
	•	ls：查看当前目录内容
	•	cd：切换目录。 

练习 2：创建几个目录并来回切换

mkdir notes
mkdir practice
mkdir test
ls

然后开始切换：

cd notes
pwd

cd ..
pwd

cd practice
pwd

cd ../test
pwd

cd ~/cli-lab/round1
pwd

这里你要体会几种写法：
	•	cd notes：进入当前目录下的 notes
	•	cd ..：回到上一级
	•	cd ../test：先回上一级，再进 test
	•	cd ~/cli-lab/round1：直接回到目标路径。Ubuntu 的新手教程就是按这种“打开终端、创建目录和文件、移动和操作文件”的顺序设计的。 

练习 3：绝对路径和相对路径

先进入：

cd ~/cli-lab/round1/notes
pwd

再分别试：

cd ..
pwd

cd ~/cli-lab/round1/practice
pwd

cd ../test
pwd

cd ~/cli-lab/round1
pwd

你要分清：
	•	绝对路径：从固定位置开始写，例如 ~/cli-lab/round1/practice
	•	相对路径：从当前所在位置开始写，例如 ../test

第 2 周：文件和目录操作

这一周练这 5 个命令：
	•	mkdir
	•	touch
	•	cp
	•	mv
	•	rm

Ubuntu 官方新手教程把“创建文件夹和文件”“移动和操作文件”作为最核心章节之一。 

练习 4：搭一个小目录结构

cd ~/cli-lab/round1
mkdir week2
cd week2
mkdir raw_data backup archive
ls

练习 5：创建测试文件

touch a.txt
touch b.txt
touch c.txt
ls

练习 6：复制文件

cp a.txt backup/
ls
ls backup

你要观察：
	•	当前目录里的 a.txt 还在
	•	backup/ 里也多了一份 a.txt

练习 7：移动文件

mv c.txt archive/
ls
ls archive

你要观察：
	•	当前目录里的 c.txt 没了
	•	archive/ 里有了 c.txt

练习 8：改名

mv b.txt notes.txt
ls

你要观察：
	•	b.txt 变成了 notes.txt

练习 9：复制并改名

cp notes.txt backup/notes_copy.txt
ls backup

练习 10：删除测试文件

先新建一个：

touch delete_me.txt
ls

再删除：

rm delete_me.txt
ls

这一步只在练习目录里做，不要碰真实资料文件。

练习 11：删除子目录中的文件

touch archive/temp.txt
ls archive
rm archive/temp.txt
ls archive

练习 14：用 less 看文件

less study_log.txt

	•	用方向键上下看
	•	按 q 退出。Apple 的说明就是这样写的。 

练习 15：只看开头

head -n 3 study_log.txt

练习 16：只看结尾

tail study_log.txt

tail -n 4 study_log.txt

练习 17：查帮助

man ls
man cp
man mv
man rm
man cat
man less

每次进去后只做三件事：
	1.	看命令名字和简介
	2.	找一两个常见参数
	3.	按 q 退出。Apple 文档明确把 man 作为查询 UNIX 命令的入口。
