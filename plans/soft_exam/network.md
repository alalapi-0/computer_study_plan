## 1. 学习边界说明

- 本文件只做学习骨架与节奏组织，不缓存具体考题或官方考纲条目。
- 软考侧重点是协议层次、常见协议功能和工程直觉；408 数值推导单独维护。

---

## 2. 建议章节清单（启动版）

1. 网络分层模型（OSI / TCP-IP 的职责边界）
2. IP 与子网基础（地址、掩码、网关的直觉）
3. TCP / UDP（连接、可靠性、端口与适用场景）
4. HTTP / HTTPS / DNS（应用层常见协议）
5. 局域网与网络设备（交换机、路由器、ARP 的基本作用）
6. 网络安全基础（认证、加密、常见风险）
7. 网络排查入口（ping、curl、ss 与浏览器 Network 面板）

---

## 3. 与 408 的差异提示

- 软考更偏“协议功能 + 场景识别”，408 更偏“字段、计算、窗口、拥塞控制推导”。
- 工程实操线的 HTTP/API 与 VPS 网络检查可以反哺本模块，但考试表达要回到标准术语。
- 网络学习容易碎片化，首轮先按“分层 → 地址 → 传输 → 应用”串起来。

---

## 4. 首轮推进建议（1 周）

- 第 1-2 天：分层模型 + IP / 子网基础
- 第 3-4 天：TCP / UDP + HTTP / DNS
- 第 5 天：网络设备 + 安全基础
- 第 6-7 天：用工程命令做一次概念回看，并整理错题来源

---

## 5. 完成当前阅读任务的最小产物

- 写出 3 个概念边界，例如“TCP vs UDP”“交换机 vs 路由器”“HTTP vs HTTPS”。
- 选定下一次要补的 1 个小节，并写清是否需要配合 Round 14 或 VPS 网络检查。
- 如果已有题目来源，只记录来源位置、错因和回炉方向，不复制题面。

---

## 6. 易错提醒（启动级）

- 看到网络题时先判断它问的是哪一层；层次错了，协议功能通常也会跟着答偏。
