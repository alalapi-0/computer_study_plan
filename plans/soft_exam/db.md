## 1. 建议章节清单（骨架，非官方考点表）

| 序号 | 章节主题 | 学习要点（概念层） |
|---|---|---|
| 1 | 数据库系统概述 | DB、DBMS、DBS；三级模式与两级映像 |
| 2 | 数据模型 | 概念模型（E-R）；逻辑模型（关系模型为主） |
| 3 | 关系代数与 SQL | 基本运算；SELECT / JOIN / 聚合 / 子查询（语法以教材为准） |
| 4 | 关系数据库设计 | 函数依赖、范式（1NF～BCNF 概念与化简思路） |
| 5 | 事务与并发控制 | ACID；封锁、死锁、隔离级别概念 |
| 6 | 数据库完整性 | 实体 / 参照 / 用户定义完整性 |
| 7 | 数据库安全与备份 | 权限、备份恢复策略概念 |
| 8 | 新技术概览 | 分布式、NoSQL 等（按当年大纲是否纳入取舍） |

> 本模块**主要为软考独有**；工程侧 SQLite 实操见 `round_11.md`，与考点笔记分开维护。

## 2. 与 408 的差异说明

| 维度 | 软考（本笔记） | 408 |
|---|---|---|
| 是否考查 | **核心模块**（上午选择题 + 下午设计题常见） | 统考408科目**一般不考**数据库系统 |
| 学习重点 | E-R 转关系、范式、SQL、事务概念 | 若考研院校复试考数据库，另建院校级计划，不写进 408 主线 |
| 与工程实操 | `round_11.md` SQLite 可作「会用」补充 | 不替代软考理论题备考 |

**结论**：数据库笔记以 `soft_exam` lane 为主，无需与 `plans/408/` 对称建文件（除非目标院校明确要求）。
