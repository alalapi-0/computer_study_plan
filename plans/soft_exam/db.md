# 软考中级（软件设计师）· 数据库模块骨架

> ⚠ 注意：最新考试大纲、题型与范围以官方发布为准：<https://www.ruankao.org.cn/>
> lane：`soft_exam`
> 关联 408：数据库通常不在 408 主体科目中，建议作为软考独立模块维护。

---

## 1. 学习边界说明

- 本文件用于学习骨架与节奏组织，不缓存具体考题及官方考纲条目。
- 各章节的深度按“可稳定得分”优先，不做教材式铺陈。

---

## 2. 建议章节清单（启动版）

1. 数据库系统基础（DB/DBMS/数据库系统概念）
2. 数据模型与关系代数基础（关系、约束、范式意识）
3. SQL 常用语句（查询、连接、聚合、分组、子查询）
4. 关系规范化（1NF/2NF/3NF/BCNF 的应用判断）
5. 事务管理（ACID、并发异常、隔离级别）
6. 索引与查询优化基础（B+树概念、执行代价直觉）
7. 备份恢复与安全（权限、完整性、审计基础）

---

## 3. 与 408 的差异提示

- 数据库是软考中的高权重模块之一，但并非 408 主体科目；需单独稳定投入。
- 软考更强调 SQL 应用与规范化判断，建议通过小题型反复巩固。
- 工程实操线中的 SQL 使用经验可迁移到本模块，但考试表达要贴近标准术语。

---

## 4. 首轮推进建议（1 周）

- 第 1-2 天：关系模型 + SQL 查询
- 第 3-4 天：规范化 + 事务并发
- 第 5 天：索引与恢复安全
- 第 6-7 天：错题归类与专题回炉

---

## 5. 完成当前阅读任务的最小产物

- 写出 3 个数据库概念边界，例如“WHERE vs HAVING”“主键 vs 外键”。
- 选定下一次要补的 1 个小节，并写清需要概念整理还是 SQL 小练习。
- 如果已经有错题或资料定位，只记录来源位置、错因和订正思路，不复制题面。

---

## 6. 易错提醒（启动级）

- 常见错误是把“分组前过滤”和“分组后过滤”混用，注意区分 `WHERE` 与 `HAVING` 的作用阶段。
