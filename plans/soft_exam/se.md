# 软考中级（软件设计师）· 软件工程模块骨架

> ⚠ 注意：最新考试大纲、题型与范围以官方发布为准：<https://www.ruankao.org.cn/>
> lane：`soft_exam`
> 关联 408：408 不单独考软件工程；本模块为软考独有

---

## 1. 学习边界说明

- 本文件仅做学习骨架，不缓存具体考题和官方考点原文。
- 章节安排用于推进节奏，不等同于官方考纲条目。

---

## 2. 建议章节清单（启动版）

1. 软件生命周期与过程模型（瀑布、迭代、敏捷等概念对比）
2. 需求工程（需求获取、分析、规格说明、变更管理）
3. 软件设计原则与架构风格（分层、微服务、事件驱动等概念框架）
4. 详细设计与模块划分（内聚、耦合、接口设计）
5. 软件测试（单元 / 集成 / 系统 / 验收；黑盒与白盒思路）
6. 软件维护与演化（ corrective / adaptive / perfective / preventive）
7. 软件项目管理基础（范围、进度、成本、质量、风险概念）
8. 软件质量保证与配置管理（版本控制、基线、审计概念）

---

## 3. 与 408 的差异提示

- 软件工程为**软考独有模块**，408 不直接考查。
- 软考侧重过程模型识别、测试方法分类、项目管理概念题；不要求完整项目实战。
- 工程实操线（`engineering` lane）中的 Git / CI / 测试实践可作为理解辅助，但答题仍以软考题型表达为准。

---

## 4. 首轮推进建议（1 周）

- 第 1-2 天：生命周期 + 需求工程
- 第 3-4 天：设计原则 + 测试分类
- 第 5 天：维护 + 项目管理概念
- 第 6-7 天：与 UML / 面向对象模块交叉复习 + 错题整理

---

## 5. 易错提醒（启动级）

- 容易混淆不同过程模型（如瀑布 vs 螺旋 vs 敏捷）的适用场景与特点，做题前先判断题目问的是"阶段顺序"还是"风险应对"还是"迭代频率"。
