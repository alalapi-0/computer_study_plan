## 1. 学习边界说明

- 本文件仅做学习骨架，不缓存具体考题和官方考点原文。
- 章节安排用于推进节奏，不等同于官方考纲条目。

---

## 2. 建议章节清单（启动版）

1. 面向对象基本概念（对象、类、消息、封装）
2. 三大特性（封装、继承、多态）及典型应用
3. 抽象类与接口（区别、使用场景）
4. 类之间的关系（关联、聚合、组合、依赖、泛化）
5. 面向对象分析与设计（OOA / OOD 基本流程概念）
6. SOLID 原则入门（单一职责、开闭、里氏替换、接口隔离、依赖倒置）
7. 常见 OOP 设计题思路（职责划分、接口抽象、扩展点识别）
8. 与 UML 类图的对应关系（概念到图的映射）

---

## 3. 与 408 的差异提示

- 软考面向对象侧重**概念识别 + 简单设计题 + UML 读图**，408 更侧重算法与系统综合。
- 软考 C 语言模块中的"面向过程"代码与本模块的 OOP 概念需区分，避免混用术语。
- 工程实操线中的 Python / Java OOP 实践可辅助理解，但答题以软考标准表述为准。

---

## 4. 首轮推进建议（1 周）

- 第 1-2 天：三大特性 + 类关系
- 第 3-4 天：抽象类 / 接口 + SOLID 原则
- 第 5 天：OOA / OOD 流程 + 简单设计题思路
- 第 6-7 天：与 UML 模块联合读图 + 错题整理

---

## 5. 易错提醒（启动级）

- 容易把"继承"和"实现接口"混为一谈；继承是 is-a 关系，实现接口是 has-capability 关系，UML 表达与代码结构不同。
