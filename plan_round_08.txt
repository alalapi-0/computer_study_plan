import sqlite3
from pathlib import Path

DB_PATH = Path("runs.db")

def init_db():
    conn = sqlite3.connect(DB_PATH)
    cur = conn.cursor()
    cur.execute("""
        CREATE TABLE IF NOT EXISTS runs (
            id INTEGER PRIMARY KEY AUTOINCREMENT,
            input_file TEXT NOT NULL,
            output_file TEXT NOT NULL,
            raw_count INTEGER NOT NULL,
            final_count INTEGER NOT NULL,
            created_at TEXT NOT NULL
        )
    """)
    conn.commit()
    conn.close()

def save_run(input_file, output_file, raw_count, final_count, created_at):
    conn = sqlite3.connect(DB_PATH)
    cur = conn.cursor()
    cur.execute("""
        INSERT INTO runs (input_file, output_file, raw_count, final_count, created_at)
        VALUES (?, ?, ?, ?, ?)
    """, (input_file, output_file, raw_count, final_count, created_at))
    conn.commit()
    conn.close()

def list_runs():
    conn = sqlite3.connect(DB_PATH)
    cur = conn.cursor()
    cur.execute("SELECT id, input_file, output_file, raw_count, final_count, created_at FROM runs ORDER BY id DESC")
    rows = cur.fetchall()
    conn.close()
    return rows

然后在 Python 里手动跑一遍：

运行：

fastapi dev api.py

FastAPI 官方 First Steps 明确给出了这条最小路径，而且 recap 里直接写了“run the development server using the command fastapi dev”。它也说明 @app.get("/") 这样的装饰器表示：这个函数负责处理某个路径上的某个 HTTP 操作，比如 GET。 

练习 7：理解最小 HTTP 语义

这一周你不用深入 HTTP，只要先记住：
	•	GET /health：拿状态
	•	GET /runs：拿一组记录
	•	以后如果要触发处理，通常会用 POST /run

MDN 的 HTTP 总览说明，HTTP 是客户端-服务器协议，用来获取资源和交换数据；HTTP request methods 文档说明，HTTP 定义了一组请求方法来表示这次请求的目的。 

第 2 周周末小成果

这一周结束时，你应该已经有：
	•	一个本地 SQLite 数据库
	•	一张最小 runs 表
	•	一个能写运行记录的工具
	•	一个最小 FastAPI 服务
	•	两个最小接口：/health 和 /runs

第 3 周：Docker 基础认知 + 最小容器化尝试 + 后续主线选择

这一周的目标不是把部署学完，而是先把容器这件事讲明白、摸一遍。Docker 官方文档把容器定义成“带齐运行所需文件的隔离进程”，多个容器共享同一个内核；镜像则是“包含运行容器所需文件、二进制、库和配置的标准化包”。Get Started 里还专门给了一个非常适合新手的演练对象：docker/welcome-to-docker，可以用它练搜索、拉取、查看镜像和镜像历史。 

练习 8：先理解容器和镜像

这周先不写 Dockerfile，先把两个概念记稳：
	•	容器：正在运行的隔离进程
	•	镜像：容器运行时所需环境的标准化包

这一步不是形式主义。后面你把 FastAPI 服务装进 Docker，本质上就是：先做镜像，再用镜像起容器。 Docker 文档就是按这个逻辑组织的。 

练习 9：如果你装了 Docker，就跑官方欢迎镜像

先搜：

docker search docker/welcome-to-docker

拉镜像：

docker pull docker/welcome-to-docker

看本地镜像：

docker image ls

看镜像历史：

docker image history docker/welcome-to-docker

Docker 官方 “What is an image?” 页面就是用 docker/welcome-to-docker 这条线来演示搜索、拉取和查看镜像层历史。 

练习 10：给你的项目写一个最小 Dockerfile

如果你想试一下，把下面这个最小版先写出来：

FROM python:3.12-slim

WORKDIR /app

COPY . /app

RUN pip install "fastapi[standard]"

CMD ["fastapi", "run", "api.py", "--host", "0.0.0.0", "--port", "8000"]

然后试着构建：

docker build -t ai-prep-tool:demo .

这一步不是因为你现在必须会部署，而是让你把“镜像是标准化包，容器是镜像的运行实例”这个概念真正落到手上。Docker 文档对镜像和容器的定义已经足够支持这一层练习。 

练习 11：给自己做一份“后续路线单选题”

这一周最后，请你强制自己写一个 next_step.md，里面只选一条主线：

第一条，数据工具 / 自动化工程主线。

第二条，后端 / API 主线。
继续补 HTTP、FastAPI、Uvicorn、SQLite，再往服务化走。FastAPI First Steps、MDN HTTP、Uvicorn 官方主页都能直接承接这条路线。

这条线你现在可以先列成副线，等前两条更稳以后再接 PyTorch 和 Hugging Face。你这轮清单里先不展开它。这个建议是路线规划，不是某一条官方文档直接规定的，但它是基于你目前项目形态做出的更稳妥排序。
