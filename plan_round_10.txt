第 1 周：拆分入口与核心逻辑

第 2 周：配置文件与日志整理

这一周的主题是：把“写死在代码里”的东西搬出来，把“散落在终端里的输出”收拢起来。

先做配置文件。你可以先用一个最小 config.ini，例如：

然后用 configparser 读取。官方文档说明它提供 ConfigParser 类，支持类似 INI 的节和键值对，而且就是为了让 Python 程序更容易被最终用户自定义。 

再做日志整理。你这一周要把“用户想看到的结果”和“程序运行痕迹”分开。logging HOWTO 明确给了一个表来说明不同场景下该用什么工具，并说明可以通过 logging.getLogger(__name__) 获取 logger，再用不同级别记录事件。也就是说，像“处理完成、输出文件路径是什么”这类可以继续给用户看，而“开始读取、过滤了多少条、去重前后数量是多少”这类就更适合交给日志系统。 

建议你这一周至少做到：
	•	logs/run.log 持久化日志文件
	•	终端也能看到简洁日志
	•	至少区分 info 和 error
	•	在异常发生时，日志里能留下足够线索

这一步会直接提升你后面排查问题的效率。logging 的官方定位就是“追踪软件运行时发生的事件”。 

第 3 周：错误处理与 CLI 设计优化

这一周的主题是：让工具在出错时更像工具，而不是直接崩掉给你一大段栈追踪。

Python 官方教程在 “Errors and Exceptions” 里明确说，运行时错误叫 exceptions，它们并不是天然致命的，程序可以捕获并处理；文档还专门讲了 try/except 和异常抛出。也就是说，你这一周的重点不是“消灭异常”，而是在对的边界上处理异常。 

我建议你重点处理这几类错误：
	•	输入文件不存在
	•	输入格式不支持
	•	配置文件缺失或配置值非法
	•	输出目录不可写
	•	数据内容不符合预期

然后，把 CLI 再优化一层。例如：
	•	合理设置默认值
	•	给每个参数写清楚 help
	•	用 choices= 限定合法格式
	•	用更清楚的参数名，少用缩写

这些都直接来自 argparse 的典型使用方式。官方文档明确说它会在参数非法时给出错误，并自动生成帮助信息，所以你越把参数定义清楚，CLI 越像一个真正给别人用的工具。

练习 3：把核心逻辑搬到 core.py

在 core.py 先只放最核心的纯函数骨架：

def filter_records(records, keyword):
    if not keyword:
        return records
    return [r for r in records if keyword in str(r)]

def dedup_records(records, fmt):
    seen = set()
    result = []
    for item in records:
        key = str(item)
        if key not in seen:
            seen.add(key)
            result.append(item)
    return result

def build_summary(input_file, output_file, raw_count, filtered_count, final_count):
    return {
        "input_file": input_file,
        "output_file": output_file,
        "raw_count": raw_count,
        "filtered_count": filtered_count,
        "final_count": final_count,
    }

这一步的目标，是让逻辑变得“可以单独调用”，而不是都塞在入口里。

练习 4：让 cli.py 调用 core.py

把 cli.py 改成最小联通版：

import argparse
from core import filter_records, dedup_records, build_summary

def parse_args():
    parser = argparse.ArgumentParser(description="AI prep tool")
    parser.add_argument("--input", required=True, help="input file path")
    parser.add_argument("--output", required=True, help="output file path")
    parser.add_argument("--format", required=True, choices=["txt", "csv", "json", "jsonl"])
    parser.add_argument("--keyword", default=None, help="keyword filter")
    parser.add_argument("--dedup", action="store_true", help="enable dedup")
    return parser.parse_args()

def main():
    args = parse_args()

    # 先用假数据跑通调用关系
    records = ["ok sample", "bad sample", "ok again"]
    raw_count = len(records)

    filtered = filter_records(records, args.keyword)
    filtered_count = len(filtered)

    final_records = dedup_records(filtered, args.format) if args.dedup else filtered
    final_count = len(final_records)

    summary = build_summary(args.input, args.output, raw_count, filtered_count, final_count)
    print(summary)

if __name__ == "__main__":
    main()

运行：

练习 8：在 log_utils.py 里整理日志初始化

新建最小日志初始化：

import logging
from pathlib import Path

def setup_logging(level="INFO"):
    log_dir = Path("logs")
    log_dir.mkdir(exist_ok=True)

    logging.basicConfig(
        level=getattr(logging, level.upper(), logging.INFO),
        format="%(asctime)s [%(levelname)s] %(message)s",
        handlers=[
            logging.FileHandler(log_dir / "run.log", encoding="utf-8"),
            logging.StreamHandler()
        ]
    )

练习 11：给 io_utils.py 补一个最小 reader 边界

例如：

from pathlib import Path
import csv
import json

def read_records(path, fmt):
    path = Path(path)

    if not path.exists():
        raise FileNotFoundError(f"Input file not found: {path}")

    if fmt == "txt":
        with path.open("r", encoding="utf-8") as f:
            return [line.strip() for line in f if line.strip()]

    if fmt == "csv":
        with path.open("r", encoding="utf-8", newline="") as f:
            return list(csv.DictReader(f))

    if fmt == "json":
        with path.open("r", encoding="utf-8") as f:
            return json.load(f)

    if fmt == "jsonl":
        rows = []
        with path.open("r", encoding="utf-8") as f:
            for line in f:
                line = line.strip()
                if line:
                    rows.append(json.loads(line))
        return rows

    raise ValueError(f"Unsupported format: {fmt}")

练习 12：在 cli.py 的边界捕获异常

例如：

from io_utils import read_records

def main():
    args = parse_args()
    config = load_config()
    log_level = config["logging"].get("level", "INFO")
    setup_logging(log_level)

    try:
        records = read_records(args.input, args.format)
    except FileNotFoundError as e:
        logging.error(str(e))
        print(f"Error: {e}")
        return
    except ValueError as e:
        logging.error(str(e))
        print(f"Error: {e}")
        return

这里的关键不是“把所有异常都抓了”，而是抓最常见、最可预期的那些。

练习 13：优化 CLI 参数定义

把参数设计再整理一遍。目标是：
	•	参数名更清楚
	•	help 更清楚
	•	尽量有合理默认值
	•	格式参数用 choices=

例如：

parser.add_argument("--input", required=True, help="path to input data file")
parser.add_argument("--output", required=True, help="path to output data file")
parser.add_argument("--format", choices=["txt", "csv", "json", "jsonl"], help="input/output data format")
parser.add_argument("--keyword", help="keep only records containing this keyword")
parser.add_argument("--dedup", action="store_true", help="remove duplicate records")

argparse 官方文档明确说：程序定义需要哪些参数，argparse 会负责解析、生成帮助信息，并在非法参数时给出错误。
