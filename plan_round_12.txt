练习 3：写 run_one_file(...)

这一层不要重写处理逻辑，直接复用你已有的函数。先做最小骨架：

from pathlib import Path
from core import filter_records, dedup_records, build_summary
from io_utils import read_records, write_records

def run_one_file(input_path, output_dir, fmt="jsonl", keyword=None, dedup=False):
    input_path = Path(input_path)
    output_dir = Path(output_dir)
    output_dir.mkdir(parents=True, exist_ok=True)

    records = read_records(input_path, fmt)
    raw_count = len(records)

    filtered = filter_records(records, keyword)
    filtered_count = len(filtered)

    final_records = dedup_records(filtered, fmt) if dedup else filtered
    final_count = len(final_records)

    output_path = output_dir / input_path.name
    write_records(final_records, output_path, fmt)

    summary = build_summary(
        str(input_path),
        str(output_path),
        raw_count,
        filtered_count,
        final_count
    )
    return summary

练习 4：写 run_batch(...)

再补一个最小批处理函数：

import json
from pathlib import Path

def run_batch(input_dir, output_dir, fmt="jsonl", keyword=None, dedup=False):
    files = scan_input_dir(input_dir, f"*.{fmt}")
    all_summaries = []
    failed = []

    for f in files:
        try:
            summary = run_one_file(f, output_dir, fmt=fmt, keyword=keyword, dedup=dedup)
            all_summaries.append(summary)
        except Exception as e:
            failed.append({"file": str(f), "error": str(e)})

    batch_summary = {
        "input_dir": str(input_dir),
        "output_dir": str(output_dir),
        "total_files": len(files),
        "success_count": len(all_summaries),
        "failed_count": len(failed),
        "failed_items": failed,
    }

    output_dir = Path(output_dir)
    output_dir.mkdir(parents=True, exist_ok=True)

    with (output_dir / "batch_summary.json").open("w", encoding="utf-8") as f:
        json.dump(batch_summary, f, ensure_ascii=False, indent=2)

    with (output_dir / "failed_items.txt").open("w", encoding="utf-8") as f:
        for item in failed:
            f.write(f"{item['file']} | {item['error']}\n")

    return batch_summary

运行可以临时在 pipeline.py 末尾加：

if __name__ == "__main__":
    result = run_batch("input/batch1", "output/batch1", fmt="jsonl", keyword="ok", dedup=True)
    print(result)

练习 6：把日志升级成 RotatingFileHandler

打开你的 log_utils.py，把文件日志部分改成类似这样：

import logging
from pathlib import Path
from logging.handlers import RotatingFileHandler

def setup_logging(level="INFO"):
    log_dir = Path("logs")
    log_dir.mkdir(exist_ok=True)

    logger = logging.getLogger()
    logger.setLevel(getattr(logging, level.upper(), logging.INFO))
    logger.handlers.clear()

    formatter = logging.Formatter("%(asctime)s [%(levelname)s] %(message)s")

    file_handler = RotatingFileHandler(
        log_dir / "run.log",
        maxBytes=50_000,
        backupCount=3,
        encoding="utf-8"
    )
    file_handler.setFormatter(formatter)

    stream_handler = logging.StreamHandler()
    stream_handler.setFormatter(formatter)

    logger.addHandler(file_handler)
    logger.addHandler(stream_handler)

RotatingFileHandler 的官方说明就是“支持磁盘日志轮转”。

第 3 周：定时任务与归档

这一周主题是：让批处理能按计划跑，并把本次结果归档起来。crontab(5) 说明 cron 行由 5 个时间字段和一个命令组成；crontab(1) 说明用户可以安装、列出和编辑自己的 crontab；cron(8) 说明 cron 会定期检查各个 crontab，并在需要的分钟执行对应命令。shutil 则很适合做高层复制和归档。  

练习 11：写最小归档脚本

打开 scripts/archive_reports.sh，先写一个最小版：

#!/bin/sh

cd "$(dirname "$0")/.."

STAMP=$(date +"%Y%m%d_%H%M%S")
ARCHIVE_DIR="archive/$STAMP"

mkdir -p "$ARCHIVE_DIR"

cp -r output "$ARCHIVE_DIR"/output_snapshot
cp -r logs "$ARCHIVE_DIR"/logs_snapshot
cp runs.db "$ARCHIVE_DIR"/ 2>/dev/null || true

给权限：

chmod +x scripts/archive_reports.sh

先手动跑一次：

./scripts/archive_reports.sh
find archive -maxdepth 2 -type d

如果你想更 Python 化，也可以把归档写进 pipeline.py，用 shutil.copy2() 或 shutil.copytree()。shutil 官方文档就是为这类高层文件操作准备的。  

练习 12：把归档接到批处理后面

第二种，在 pipeline.py 结束后用 subprocess.run() 调 shell 脚本。subprocess.run() 会等待命令完成并返回 CompletedProcess。

import subprocess

练习 13：配置最小 cron 任务

先看当前表：

然后编辑：

crontab(5) 说明一条记录由 5 个时间字段和一个命令组成。 

为了更稳一点，你也可以让 cron 直接执行归档版流程，例如：

35 21 * * * /bin/sh /Users/你的用户名/cli-lab/round7/final_lab/ai_prep_tool/scripts/archive_reports.sh

练习 14：写清楚 cron 的工作目录问题

这一条很重要：cron 只是在某个时间执行命令，它不会自动帮你切到项目目录，所以你最好在 shell 脚本里显式 cd 到仓库根目录。crontab(5) 只负责时间与命令匹配，不负责帮你“进入正确目录”。
