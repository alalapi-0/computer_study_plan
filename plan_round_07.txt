练习 2：分别写四个 reader

把 read_formats.py 补成下面这个结构：

from pathlib import Path
import csv
import json

def read_txt_lines(path: Path):
    with path.open("r", encoding="utf-8") as f:
        return [line.strip() for line in f if line.strip()]

def read_csv_rows(path: Path):
    with path.open("r", encoding="utf-8", newline="") as f:
        return list(csv.DictReader(f))

def read_json_file(path: Path):
    with path.open("r", encoding="utf-8") as f:
        return json.load(f)

def read_jsonl_file(path: Path):
    rows = []
    with path.open("r", encoding="utf-8") as f:
        for line in f:
            line = line.strip()
            if line:
                rows.append(json.loads(line))
    return rows

base = Path("input")

print("txt:", read_txt_lines(base / "sample.txt"))
print("csv:", read_csv_rows(base / "sample.csv"))
print("json:", read_json_file(base / "sample.json"))
print("jsonl:", read_jsonl_file(base / "sample.jsonl"))

这里你会同时练到几件很关键的事：Path.open()、csv.DictReader()、json.load()、以及逐行 json.loads() 读取 .jsonl。这些做法都直接对应官方文档建议的用法。

练习 1：做最小 CLI

在 filter_tool.py 里先写参数解析：

import argparse

parser = argparse.ArgumentParser(description="Simple AI prep tool")
parser.add_argument("--input", required=True, help="input file path")
parser.add_argument("--output", required=True, help="output file path")
parser.add_argument("--format", required=True, choices=["txt", "csv", "json", "jsonl"])
args = parser.parse_args()

print(args)

运行：

python3 filter_tool.py --input input/sample.txt --output output/out.txt --format txt

再试：

python3 filter_tool.py --help

如果能自动看到 help，说明你已经把 CLI 的最小骨架搭起来了。argparse 官方文档对这正是它的核心卖点。 

练习 2：加关键词过滤

继续给参数加两个选项：

parser.add_argument("--keyword", help="keyword filter", default=None)
parser.add_argument("--dedup", action="store_true", help="enable dedup")

然后在读取后，做最简单的过滤逻辑。

如果是 txt，你可以直接按整行是否包含关键词过滤。
如果是 jsonl 或 csv，你可以先把整条记录转成字符串后再判断，或者先约定只检查 label / text 字段。

示例调用：

python3 filter_tool.py --input input/sample.jsonl --output output/out.jsonl --format jsonl --keyword ok

这一练的关键不是“过滤写得多优雅”，而是让你开始把“读取 → 处理 → 输出”做成一条完整链路。这里用到的仍然是 argparse 和标准库文件读写能力。 

练习 3：加去重

如果是 txt，直接按整行去重。
如果是 jsonl，你可以先按整条 JSON 字符串去重，或者如果有 id 字段，就按 id 去重。

最简单的思路：

seen = set()
result = []

for item in records:
    key = str(item)
    if key not in seen:
        seen.add(key)
        result.append(item)

这里就是在实战里使用你前面学过的 set 思维。虽然这个做法本身是你代码设计的一部分，不是官方 API 规定，但它正好建立在 Python 内置容器的常规使用上。 

练习 4：加日志

先写一个最小 logging 配置：

import logging
from pathlib import Path

log_dir = Path("logs")
log_dir.mkdir(exist_ok=True)

logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s [%(levelname)s] %(message)s",
    handlers=[
        logging.FileHandler(log_dir / "run.log", encoding="utf-8"),
        logging.StreamHandler()
    ]
)

logging.info("tool started")

然后在关键步骤打日志：
	•	读取前
	•	读取后
	•	过滤后
	•	去重后
	•	输出完成后

logging 模块文档说明它提供灵活的事件日志系统；HOWTO 说明了 debug / info / warning / error / critical 的常见级别，而参考文档也说明 basicConfig() 能处理很多常见场景。

练习 1：统一 reader / writer

别再把每种格式的处理写得很散。把代码整理成这几个层次：
	•	read_records(path, fmt)
	•	filter_records(records, keyword)
	•	dedup_records(records, fmt)
	•	write_records(records, path, fmt)
	•	build_summary(...)

这里最重要的是“统一接口”。
例如：

records = read_records(input_path, fmt)
filtered = filter_records(records, keyword)
cleaned = dedup_records(filtered, fmt) if args.dedup else filtered
write_records(cleaned, output_path, fmt)

这种结构会让你的脚本第一次有“工具感”，而不是一长串乱逻辑。

你这一轮最容易踩的坑

第一个坑，是一上来就上 pandas。
你当然以后可能会用到它，但这一轮先不需要。因为你现在最重要的是把“文件 → 记录 → 处理 → 输出”这条基础链路彻底做通，而 Python 标准库已经足够覆盖这条链路。 

第二个坑，是把 .json 和 .jsonl 混了。
普通 .json 往往是一个完整对象或数组，而 .jsonl 强调“每行一个合法 JSON 值”，读取方式通常是逐行处理。 

第三个坑，是所有状态都用 print()。
print() 适合简单直接输出，但需要长期保留、分级和排查的问题，更适合交给 logging。官方 HOWTO 对这一点说得很明确。 

第四个坑，是路径全靠字符串拼。
这会让目录切换、输出路径生成、跨平台运行都更容易出错，而 pathlib 的设计目的就是让这件事更稳。
