Round 0 的定位

这一轮不是正式学 Linux 全套，也不是学编程。它只解决一件事：

先把你的学习系统搭起来，让你第一次对终端产生“我能用它做点事”的感觉。

MIT 的 Missing Semester 本来就是一门专门补“学校通常不系统教、但实际做项目非常重要的工具能力”的课程，2026 年课程表里第一讲就是 Course Overview + Introduction to the Shell。Apple 的 Terminal 指南则把 Terminal 定义为用命令行执行命令、运行工具和脚本的应用入口。 

为什么 Round 0 仍然要单独保留

对你现在这种节奏来说，最危险的不是学得慢，而是一上来信息太多。Ubuntu 的官方新手教程明确写了它假设学习者没有先验知识，目标是让你对几个基础命令和概念先熟起来；Ubuntu 教程列表里还标了这篇《The Linux command line for beginners》的难度是 1/5。这正适合作为 Round 0 和 Round 1 之间的缓冲入口。 

Round 0 学什么

这一轮只学这 4 件事：
	1.	Terminal 和 shell 大概分别是什么
	2.	怎么打开终端、怎么在里面输入命令
	3.	最基础的几个命令：pwd、ls、cd、mkdir、touch、cat、man
	4.	建立一份你自己的命令小抄和练习目录

Apple 官方入门页直接用 ls 和 man 做最基础示例；Linux Journey 的 File System Navigation 页面则把 ls、cd、pwd 和目录导航单独作为最前面的内容来讲。 

Round 0 不学什么

这一轮先不碰：
	•	Git
	•	管道和重定向
	•	shell 脚本
	•	权限细节
	•	sudo
	•	数据结构和算法本体

原因很简单。Missing Semester 2026 的课程安排里，shell 入门、命令行环境、Git、本地开发工具这些本来就是分讲次推进的；你现在只需要把第一个台阶站稳，不需要把后面的内容提前塞进来。

1. Round 0 你这一轮要练到什么程度

你这轮不需要学会很多命令。你只需要做到这几件事：
	•	会打开终端
	•	知道 Terminal 和 shell 不是一回事
	•	会用 pwd、ls、cd、mkdir、touch、cat、man
	•	有一个自己的练习目录
	•	有一份自己的命令小抄

Apple 说明得很清楚：Terminal 是应用，里面每个窗口都代表一个 shell 进程；窗口里的提示符表示你现在可以输入命令。MIT Missing Semester 也明确说，shell 的核心作用是运行程序、给程序输入、查看程序输出。  

⸻

2. 先打开终端

如果你在 Mac 上，按这两种方式都可以：

第一种，用 Spotlight 搜索 “Terminal”。MIT Missing Semester 2026 的 shell 讲义把这列为 macOS 的一种打开方式。  

第二种，用 Finder 进入：

/Applications/Utilities

然后双击 Terminal。Apple 官方就是这样写的。  

打开之后，你会看到一个提示符，类似：

michael@MacBook-Pro ~ %

Apple 解释说，这类提示符通常会包含主机名、当前工作文件夹、用户名和提示符号；其中 ~ 常表示 home 文件夹。  

⸻

3. 先建 Round 0 总练习目录

先不要乱练，先把你的练习区建出来。

cd ~/cli-lab/round0
pwd
ls

这里你只要记住三件事：
	•	mkdir -p：创建目录
	•	cd：进入目录
	•	pwd：看你现在在哪

Ubuntu 的新手教程也把“打开终端”“创建文件夹和文件”放在最前面的几个步骤里，而且特别强调绝对路径和相对路径很重要。

	•	打开终端
	•	进入一个目录
	•	创建目录
	•	创建文件
	•	看文件
	•	查命令说明

这和 Ubuntu 新手教程那条主线是对齐的：先打开 terminal，再创建目录和文件，再逐步熟悉 shell。
